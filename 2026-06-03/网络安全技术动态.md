## Rapid7 披露 Gogs 认证后参数注入 RCE，Metasploit 模块已提供且厂商暂未修复

**入选理由：** Rapid7 Labs 披露 Gogs 参数注入 RCE，影响自托管 Git 服务；文章明确称无补丁、可自动化利用，并提供 Metasploit 模块，属于高优先级攻防工具/重大漏洞动态。

**技术细节：**

- 漏洞类型：CWE-88 argument injection；Rapid7 评分 CVSS v4 9.4 Critical；暂未见正式 CVE 编号，Rapid7 披露关联 GHSA-qf6p-p7ww-cwr9。
- 影响版本：Rapid7 确认 Gogs 0.14.2 与 0.15.0+dev commit b53d3162 受影响；所有支持 “Rebase before merging” 的早期版本可能受影响。
- 触发条件：任意认证用户创建 PR，使用恶意 base branch name 注入 `git rebase --exec`；Gogs `Merge()` 在 `internal/database/pull.go` 中直接把 `pr.BaseBranch` 传给 `git rebase`，未加 `--` 或 `--end-of-options`。
- 默认攻击路径：Gogs 默认开放注册且默认不限制仓库创建时，攻击者可注册账号、创建仓库、开启 rebase merging，然后自包含完成利用；无需管理员权限，也无需其他用户交互。
- 利用链：恶意分支名如 `--exec=touch${IFS}/tmp/rce_proof` 可被 Git 当作 `--exec` 参数处理；Linux 可通过 base64 inline payload，Windows 需文件型 payload。`Merge()` 第 3 步 `git rebase` 已触发 RCE，即使后续 checkout 失败并返回 HTTP 500。
- 影响：以 Gogs server process user 执行任意命令，可读取所有仓库、凭据、API token、SSH key、2FA secrets，并可篡改托管代码。Rapid7 称 Shodan 检索到约 1,141 个互联网暴露 Gogs 登录实例。
- 攻防工具：Rapid7 提供 Metasploit module，支持 Linux/Windows 和 own_repo / existing_repo 两种模式。

**影响与处置：** 在无补丁状态下，应立即关闭公开注册（`DISABLE_REGISTRATION=true`）、限制仓库创建（`MAX_CREATION_LIMIT=0` 或按用户限制）、降低 rebase merging 暴露面，并检查是否存在以 `--` 开头的分支、失败 PR、仓库 git 状态异常、HTTP 500 merge 日志、`msf_<hex>` API token、异常 payload 文件。多租户 Gogs 实例风险最高，应考虑临时隔离或迁移到已修复平台。

**信息来源：**

- Rapid7 分析：https://www.rapid7.com/blog/post/ve-authenticated-rce-via-argument-injection-gogs-unfixed/
- Metasploit PR：https://github.com/rapid7/metasploit-framework/pull/21515
- Gogs 项目：https://github.com/gogs/gogs

## 开发者工具供应链：Red Hat npm 包 Miasma 事件仍在升温，需与 TanStack/Nx 统一排查凭据面

**入选理由：** 多个安全媒体和厂商在过去 24 小时继续报道 Miasma / Shai-Hulud 变体感染 Red Hat `@redhat-cloud-services` npm 包，命中 package registry compromise、CI/CD credentials、developer secrets 固定监测项。当前检索结果主要为安全媒体与厂商解读，尚未取得 Red Hat 官方逐包公告，故作为供应链候选提醒，不与 CISA KEV 条目合并。

**技术细节：**

- 公开报道指向 30+ 或 32 个 `@redhat-cloud-services` scope npm 包被发布恶意版本，目标包括 CI/CD、云凭据和开发者密钥。
- 报道称该变体沿用 Shai-Hulud 类自传播思路，增加对 Google Cloud、Azure 身份或云凭据采集的关注。
- 与 TanStack 事件的共同风险面包括 npm install lifecycle scripts、包维护者身份、CI runner 可读凭据、lockfile/私有镜像缓存、撤回后残留安装路径。

**影响与处置：** 使用 `@redhat-cloud-services/*` 的团队应等待官方逐包版本清单并先做保守处置：冻结依赖、检查 lockfile、清理 CI cache、关闭 install scripts 做离线 tarball 检查、轮换在相关 runner/开发机上可访问的 npm/GitHub/cloud/Vault/Kubernetes 凭据。因缺少官方版本清单，本条不列具体受影响版本，避免误报。

- HeroDevs 分析：https://www.herodevs.com/blog-posts/miasma-npm-worm-steals-cloud-creds-and-hijacks-ci-cd
