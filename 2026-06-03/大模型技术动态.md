# 大模型技术动态日报（2026-06-03）

## Microsoft AI 发布 MAI-Thinking-1 技术报告，披露 35B active / 1T total MoE 推理模型

**入选理由：** Microsoft AI 于 2026 年 6 月 2 日发布《MAI-Thinking-1: Building a Hill-Climbing Machine》技术报告，属于头部厂商对新推理模型训练体系、架构、数据和评测的实质披露。

**技术细节：**

- 模型规模：MAI-Thinking-1 基于 MAI-Base-1，报告称为 35B active / 1T total sparse MoE；基础模型在 8K GB200 GPU 的 Azure 集群上从零训练。
- 训练数据：预训练主阶段约 30T tokens，随后进行约 3.55T tokens 的 mid-training；报告强调不使用第三方模型蒸馏数据，训练语料来自公开与授权的人类生成数据，并做去重、PII 风险过滤和安全过滤。
- 架构：decoder-only Transformer；周期性 local/global attention；MoE 与 dense FFN 交替；top-8/512 experts；采用 o200k_base tokenizer；mid-training 后最大上下文长度为 256K。
- 强化学习：从无推理轨迹暴露的 checkpoint 开始做 RL climb，分别训练 STEM reasoning、agentic coding/tool use、helpfulness/safety 三类 specialist，再通过 SFT 合并并进行最终 RL；报告给出 GRPO、adaptive entropy control、self-distillation、长输出训练等细节。
- 公开评测：报告列出 AIME 2025 97.0%、AIME 2026 94.5%、LiveCodeBench v6 87.7%、SWE-Bench Verified 73.5%、SWE-Bench Pro 52.8%、GPQA Diamond 84.2%。agentic coding 评测使用 256K 总上下文，SWE-Bench 相关任务启用 bash 与 string-replace 工具。
- 安全：报告披露了 internal safety evaluation、red teaming、安全/帮助性权衡、TAP 类自适应攻击和低资源语言绕过的缓解训练，但危险能力/能力提升类结构化评估不在本次报告范围内。

**影响与处置：** 这是 Microsoft AI 对自主推理模型训练管线的系统级披露，值得重点关注其“非蒸馏 + MoE + 长上下文 + 工具使用 RL”的组合。由于报告暂未给出模型 API、权重、价格或可用渠道，工程侧应先把它视为技术报告而非可直接迁移的生产模型；后续重点跟踪 Azure AI Foundry / Copilot / Bedrock 之外的 Microsoft AI 渠道是否开放 MAI-Thinking-1 或衍生模型。

**信息来源：**

- Microsoft AI 技术报告 PDF：https://microsoft.ai/wp-content/uploads/2026/06/main_20260602_2.pdf

## OpenAI frontier models 与 Codex 在 AWS 一般可用，涉及 Bedrock、GovCloud 与企业治理链路

**入选理由：** OpenAI 于 2026 年 6 月 1 日公告 OpenAI frontier models 与 Codex 在 AWS 上一般可用，虽不属于新模型发布，但属于头部模型能力的云平台可用性变化。

**技术细节：**

- 可用渠道：OpenAI models on Amazon Bedrock；Codex on Amazon Bedrock。
- 覆盖环境：公告称覆盖 Commercial 与 GovCloud regions。
- 能力范围：OpenAI models on Amazon Bedrock 面向 AWS-native security/governance controls 下的 AI 应用构建；Codex on Amazon Bedrock 面向代码编写、审查、调试和现代化改造。
- 企业侧变化：采购、账单、安全审查、合规与治理流程可沿用 AWS 侧已有路径。公告未披露新的模型 ID、价格、上下文长度或 API 速率限制。
- 安全相关路线：OpenAI 提到未来将通过 AWS 提供 Daybreak 相关能力，包括 cyber models、Codex Security、安全代码审查、威胁建模、补丁验证、依赖风险分析、检测与修复建议。

**影响与处置：** 对已深度使用 AWS 的企业，OpenAI 模型与 Codex 的落地阻力降低，尤其是需要 GovCloud、集中采购和统一审计的环境。落地前应复核 Bedrock 上具体可用模型 ID、区域、数据保留策略、日志字段、VPC/私网接入、IAM 边界、模型调用权限和 Codex 可执行工具范围；不要把 OpenAI 官网公告等同于所有区域、所有账号立即可用。

**信息来源：**

- OpenAI 官方公告：https://openai.com/index/openai-frontier-models-and-codex-are-now-available-on-aws/
- AWS OpenAI on Bedrock 页面：https://aws.amazon.com/bedrock/openai/
