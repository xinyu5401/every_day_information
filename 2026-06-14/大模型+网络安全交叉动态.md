# 2026-06-14 大模型+网络安全交叉动态

## 1. Anthropic 紧急下线 `Claude Fable 5` / `Claude Mythos 5`：事件核心转向“模型防护、越狱证据与出口管制”

**入选理由**  
这是过去 24 小时内最值得关注的头部模型安全事件之一：不是普通停机，而是官方确认因政府指令暂停旗舰/准旗舰模型访问，理由与潜在越狱能力有关。

**技术细节**  
- Anthropic 于 **2026-06-12** 发布公告称，美国政府以国家安全权限下达指令，要求暂停 `Fable 5` 与 `Mythos 5` 的所有访问；Anthropic 记录收到指令的时间为 **2026-06-12 17:21 ET**。
- 官方表述显示，政府担忧点在于一种可绕过 `Fable 5` 安全防护的“非通用越狱”方法；Anthropic 则称其验证后只发现少量此前已知、且较轻的漏洞发现能力，并称其他公开模型也能做到。
- 这两款模型在 **2026-06-09** 刚发布：`Fable 5` 面向通用用户，`Mythos 5` 通过 `Project Glasswing` 面向受信任的网络防御者。Anthropic 当时披露的价格为 **$10 / 百万输入 token、$50 / 百万输出 token**。
- 发布页还写明，`Fable 5` 在部分高风险网络安全请求上会回退到 `Claude Opus 4.8`；Anthropic 称该类保护平均触发比例低于 **5%** 会话，并要求 `Fable` 保留 **30 天** 客户数据用于越狱监测与缓解。

**影响与处置**  
- 对依赖 `Fable 5`/`Mythos 5` 的代码、Agent 或安全分析流程，这不是普通模型升级，而是硬性可用性中断；需要立即准备备用模型和失败切换策略。
- 这次事件也把“模型能否被越狱”从实验室问题推到了合规与出口管制层面，后续所有高能力代码/安全模型都可能被要求提供更强的审计、监测和回滚能力。
- 对内部 Agent 系统，建议补齐最少基线：工具白名单、审批开关、外部内容到工具参数的传播记录、敏感任务回放日志，以及模型切换时的安全回归测试。

## 2. MAStrike 进入 arXiv 新稿：把多智能体“串谋式”攻击红队化，并要求基于完整执行轨迹评估

**入选理由**  
论文于 **2026-06-12** 进入 `cs.CR` recent，直接命中 `LLM-MAS`、agent tracing 与 guardrail 评测，是交叉领域里质量较高的新信号。

**技术细节**  
- 论文：`MAStrike: Shapley-Guided Collusive Red-Teaming on Multi-Agent Systems`，arXiv: `2606.12918`。
- 研究目标不是单代理 prompt injection，而是多代理系统中的协同行为攻击，强调多个代理之间的通信、任务拆分和信息流会共同构成攻击面。
- 论文摘要页显示，作者将完整 MAS 执行轨迹作为输入，轨迹包含中间代理消息、工具调用和最终决策，再交给轨迹级 guardrail 做事后安全判定。
- 公开摘要还提到，实验覆盖 **3** 类协同攻击场景、累计 **1000+** 轮对话，重点是量化各代理在不安全结果中的边际贡献。

**影响与处置**  
- 这类研究的现实价值在于提醒团队：多代理系统的安全单元不再只是“单条提示词”，而是完整执行轨迹。
- 若系统已经存在 planner / executor / reviewer / browser 之类角色分工，应补日志字段，至少保留代理间消息、工具入参与出参、审批决策、最终高风险动作。
- 对 MCP、多代理代码 Agent 与浏览器 Agent，后续更可能出现“多角色串谋”型攻击评测，而不是只看单节点拒答率。

**信息来源**  
- https://arxiv.org/abs/2606.12918
- https://arxiv.org/list/cs.CR/recent
