# 2026-06-14 大模型技术动态

## 1. OpenAI 调整 ChatGPT 模型池：GPT-5.2 退场，GPT-5.4 mini 进入回退链路

**入选理由**  
OpenAI 官方 `ChatGPT — Release Notes` 在最近 20 小时内更新，属于面向生产用户的实质性模型可用性调整，不是营销稿。

**技术细节**  
- 自 **2026-06-12** 起，`GPT-5.2 Instant`、`GPT-5.2 Thinking`、`GPT-5.2 Pro` 不再在 ChatGPT 中可选。
- 既有对话会自动续接到对应的 `GPT-5.5` 模型，说明 OpenAI 已把 5.5 系列作为 ChatGPT 主力承接层。
- `GPT-5.4 mini` 正在进入 ChatGPT：对 Free/Go 用户作为 “Thinking” 能力入口，对其他付费用户则作为 `GPT-5.4 Thinking` 的限额回退模型。
- 同次更新还写明：`GPT-5.5 Instant` 与 `GPT-5.5 Thinking` 不再提供 canvas，写作和编码改由聊天中的文本块、代码块直接承接。

**影响与处置**  
- 依赖 ChatGPT 旧模型行为的团队需要重新做提示词、输出风格和工具流回归，尤其是从 5.2 切到 5.5 的长对话兼容性。
- 若内部 SOP 仍引用 `GPT-5.2`，应尽快更新为 `GPT-5.5` 或 `GPT-5.4 mini` 回退逻辑，避免培训文档与实际产品行为脱节。
- 这次调整仅确认影响 ChatGPT 端；若业务同时用 API，需要单独核验 API 模型可用性与退役计划。

**信息来源**  
- https://help.openai.com/en/articles/6825453-chatgpt-release-notes

## 2. HyperTool 提出 MCP 风格“复合工具调用”接口，显著抬升多步工具使用准确率

**入选理由**  
该论文于 **2026-06-11** 进入 arXiv 新稿列表，直接命中 MCP/tool-use 主线，并给出明确的真实 MCP 环境评测增益。

**技术细节**  
- 论文：`HyperTool: Beyond Step-Wise Tool Calls for Tool-Augmented Agents`，arXiv: `2606.13663`。
- 核心思路不是继续做“单步工具调用”，而是把一段可执行代码块包装成一次外层调用，让模型在一次调用内完成多工具编排、变量传递和中间结果处理。
- 作者将其描述为统一的“可执行 MCP-style tool interface”，本质上是在模型可见轨迹外折叠局部确定性子流程，降低上下文被低层数据流占满的问题。
- 在 `MCP-Universe` 上，`Qwen3-32B` 平均准确率从 **15.69%** 提升到 **35.29%**，`Qwen3-8B` 从 **9.93%** 提升到 **33.33%**；论文同时称其平均准确率超过 `GPT-OSS` 和 `Kimi-k2.5`。

**影响与处置**  
- 对需要多步外部工具链的 Agent 框架，这是比“继续强化函数调用 prompt”更偏系统层的改造方向。
- 但把多步过程折叠进一次执行，也会压缩中间审计面；工程落地时应同步补执行日志、输入输出快照和 provenance 字段，否则排障与安全审计会变差。
- 值得继续观察是否会带动 MCP 生态从“单步 schema 调用”转向“脚本化复合调用”。

**信息来源**  
- https://arxiv.org/abs/2606.13663
- https://arxiv.org/list/cs.CL/recent
