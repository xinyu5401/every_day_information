# 2026-06-14 网络安全技术动态

## CISA KEV 到期窗口

### 1. Ivanti Sentry `CVE-2026-10520` 进入当日处置窗口

**入选理由**  
CISA KEV 明确命中 **Due Date = 2026-06-14**；这不是“新上榜”新闻，而是当天必须完成处置的在野利用窗口。

**技术细节**  
- `CVE-2026-10520`：Ivanti Sentry OS Command Injection，`CVSS 10.0`。
- Ivanti 官方称受影响版本为修复前版本，修复版本为 `R10.5.2`、`R10.6.2`、`R10.7.1` 及以上。
- Rapid7 指出该漏洞可经未认证端点 `/mics/api/v2/sentry/mics-config/handleMessage` 触发，攻击者通过提交 `message` 参数，最终达成以 `root` 身份执行任意 OS 命令。
- CISA KEV 信息显示：`Date Added = 2026-06-11`，`Due Date = 2026-06-14`，说明今天就是 FCEB 基线要求的最后期限。

**影响与处置**  
- Ivanti Sentry 是移动设备到后端系统之间的网关，通常属于互联网暴露或边界邻近资产，优先级应高于普通内网应用。
- 仍在运行受影响版本的团队应立即升级到修复版本，并复核是否存在异常管理账户、可疑 `POST` 请求和配置下发痕迹。
- 若短时无法升级，应最少先做访问面收缩、外部暴露清点和日志回溯。

### 2. Oracle PeopleSoft `CVE-2026-35273` 进入未来 24 小时处置窗口

**入选理由**  
CISA KEV 检索结果显示该漏洞 `Due Date = 2026-06-15`，已落入“未来 24 小时到期窗口”；同时 Rapid7 在昨日确认在野利用。

**技术细节**  
- `CVE-2026-35273`：Oracle PeopleSoft PeopleTools `Updates Environment Management` 组件未认证远程代码执行，`CVSS 9.8`。
- Oracle 官方确认受影响版本为 `PeopleTools 8.61`、`8.62`，攻击面为 `HTTP`，且“remote exploitable without authentication”。
- Oracle 风险矩阵进一步写明：成功利用可导致对 `PeopleSoft Enterprise PeopleTools` 的完全接管。
- CISA 于 **2026-06-12** 将其加入 KEV；检索摘要显示 `Due Date = 2026-06-15`，已进入明日到期窗口。

**影响与处置**  
- PeopleSoft 属典型关键业务系统，且这次打到的是 `PeopleTools` 管理/更新链路，业务中断和横向移动风险都偏高。
- 若仍暴露 `PSEMHUB` 或相关更新管理组件，应优先套用 Oracle 安全告警中的补丁/缓解说明，并验证对应服务是否已停用或受限。
- 这类漏洞不适合等常规补丁窗口；应按紧急变更处理，并检查是否已有异常 Web 请求、计划任务或应用层落地文件。

## 1. Oracle PeopleSoft `CVE-2026-35273`：Rapid7 确认在野利用，Oracle 已发紧急告警

**入选理由**  
Rapid7 于 **2026-06-13** 发布“Active Exploitation”分析；Oracle 已在 **2026-06-10** 罕见通过 Security Alert 通道单独下发修复，说明紧急程度高。

**技术细节**  
- 漏洞编号：`CVE-2026-35273`，组件：`Updates Environment Management`。
- Oracle 官方定级 `CVSS 9.8`，向量为 `AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H`。
- Oracle 明确该漏洞可经网络未认证利用，成功后可导致远程代码执行；风险矩阵文本版进一步表述为可“takeover” PeopleTools。
- CISA 已将其列入 KEV，证明不是单纯研究披露，而是已有实战利用证据。

**影响与处置**  
- 受影响对象是关键 ERP/人事/财务栈常见组件，若对公网或第三方网络暴露，优先级应视同高危边界应用。
- 需要同时做三件事：打补丁/套缓解、核验 `PSEMHUB` 与更新链路暴露、做入侵痕迹排查。
- 如环境存在不再受支持的旧版 PeopleTools，应尽快规划升级；Oracle 明确仅对受支持版本提供此类安全告警修复。

## 2. Jenkins 核心安全通告修复反序列化、存储型 XSS 与 `config.xml` 明文秘密暴露

**入选理由**  
虽然不是在野利用事件，但影响对象是 Jenkins 核心而非边缘插件，且包含可升级到脚本控制台代码执行的高危链条，具备广泛基础设施影响。

**技术细节**  
- Jenkins 安全通告日期：**2026-06-10**。
- 核心高危项是 `SECURITY-3707 / CVE-2026-53435`：攻击者可通过受控 `config.xml` 提交让 Jenkins 反序列化任意核心或插件类型，随后处理 HTTP 请求。
- 官方说明该问题可被用于冒充任意用户发起请求，最高可到脚本控制台执行任意代码，也可读取 Jenkins 控制器上的任意文件。
- 另一个值得关注的供应链侧问题是 `SECURITY-3744 / CVE-2026-53442`：`POST config.xml` 中的明文秘密会原样落盘，并在后续 `GET config.xml` 中再次暴露给具备 `Item/Extended Read` 权限的用户。
- 受影响版本：Jenkins weekly `<= 2.567`，LTS `<= 2.555.2`；修复版本为 `2.568` 和 `2.555.3`。

**影响与处置**  
- Jenkins 往往掌握 CI/CD 凭据、制品签名、部署密钥和云访问令牌，`config.xml` 面的漏洞不能按普通中后台漏洞处理。
- 除升级外，应审计谁拥有 `Overall/Read`、`Item/Extended Read`、`Agent/Configure`、`POST config.xml` 相关权限，并回溯敏感作业/节点配置是否被异常读取或提交。
- 若组织内使用 Jenkins 作为制品发布入口，建议同步轮换高价值凭据，避免历史明文秘密已被读取后继续有效。

**信息来源**  
- https://www.jenkins.io/security/advisory/2026-06-10/
