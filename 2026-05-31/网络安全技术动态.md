# 网络安全技术动态日报（2026-05-31）

## 1. PAN-OS GlobalProtect CVE-2026-0257：认证绕过已出现有限在野利用并进入 CISA KEV

**入选理由：** Palo Alto Networks 5 月 29 日更新公告，将漏洞利用成熟度标为 `ATTACKED`；Rapid7 同日报告已观测到利用，且称 CISA 已将其加入 KEV。

**技术细节：** CVE-2026-0257 是 PAN-OS GlobalProtect portal/gateway 的认证绕过漏洞，CVSS-BT/CVSS-B 7.8，高危。触发条件较具体：GlobalProtect portal 或 gateway 启用 Authentication Override cookies，且认证覆盖 cookie 证书与 HTTPS 服务存在特定共享配置。成功利用可绕过安全限制建立未授权 VPN 连接。受影响版本覆盖 PAN-OS 10.2、11.1、11.2、12.1 多个分支以及 Prisma Access 对应版本；Panorama 和 Cloud NGFW 不受影响。

**影响与处置：** 暴露 GlobalProtect 的组织应优先核查是否启用 authentication override cookie，并确认是否复用了证书。处置动作包括升级到公告列出的修复版本，或临时禁用 Authentication Override，或为 Authentication Override cookies 使用专用证书。升级后 GlobalProtect 用户需要重新认证一次。SOC 应检查异常 VPN 会话、未知源 IP 的认证绕过迹象和后续横向移动。

**信息来源：**

- https://security.paloaltonetworks.com/CVE-2026-0257
- https://www.rapid7.com/blog/post/etr-rapid7-observed-exploitation-of-pan-os-globalprotect-authentication-bypass-vulnerability-cve-2026-0257/
- https://www.cisa.gov/known-exploited-vulnerabilities-catalog

## 2. Jenkins 2026-05-27 安全公告：LDAP/AD 插件 RCE 链与多项文件读取、路径穿越问题

**入选理由：** Jenkins 官方 5 月 27 日披露多插件漏洞，涉及 CI/CD 核心控制面，部分问题可导致 Jenkins controller RCE 或敏感文件读取。

**技术细节：** LDAP Plugin `807.v7d7de30930cf` 及更早版本会跟随 LDAP referrals，攻击者可将 referral 指向 RMI URL，诱导 Jenkins 反序列化攻击者控制的数据；对应 CVE-2026-48916（SSRF）和 CVE-2026-48917（deserialization）。Active Directory Plugin `2.41` 及更早版本存在类似问题，对应 CVE-2026-48918 和 CVE-2026-48919。Email Extension Plugin CVE-2026-48920 可通过 `data-inline` 图片 URL 读取 controller 文件；Pipeline: Groovy Libraries Plugin CVE-2026-48921 未禁止 shared library 符号链接；Credentials Binding Plugin CVE-2026-48922 对 file/zip 凭据文件名清理不充分，特定低权限配置下可能写入节点任意位置并扩展为 RCE。

**影响与处置：** Jenkins 管理员应按公告升级相关插件：LDAP `807.809.vd3a_4e5e4ec98`、Active Directory `2.41.1`、Credentials Binding `725.ve52b_2328a_fde`、Email Extension `1933.1935.v276319e3cc47`、Pipeline Groovy Libraries `798.v5cc688825312` 等。无法立即升级 Active Directory Plugin 的实例，可临时设置 `hudson.plugins.active_directory.referral.ignore=true`。同时检查低权限用户是否可配置凭据、shared libraries 和邮件内容。

**信息来源：**

- https://www.jenkins.io/security/advisory/2026-05-27/
- https://www.jenkins.io/security/advisories/

## 3. Nx Console v18.95.0 供应链事件：VS Code 扩展短时投毒后被列入 KEV

**入选理由：** NVD 条目和 CISA KEV 信息显示 CVE-2026-48027 已进入已知在野利用治理范围；Nx 官方 postmortem 给出攻击时间线与根因。

**技术细节：** 攻击者在 2026-05-18 12:30 至 13:09 UTC 期间，将恶意 Nx Console `v18.95.0` 发布到 Visual Studio Marketplace 和 Open VSX。Nx 官方称攻击者在 5 月 11 日窃取凭据后，曾在 GitHub 仓库中活动约 7 天而未被发现。StepSecurity 分析称恶意扩展面向开发者凭据、云基础设施 token 和 CI/CD secrets。NVD 将该问题登记为 CVE-2026-48027，并引用 GitHub/Nx/StepSecurity 资料。

**影响与处置：** 使用 Nx Console 且开启 VS Code 自动更新的团队，应假设安装窗口内开发机可能泄露凭据。优先动作是卸载恶意版本、升级到官方修复版本、轮换 GitHub/GitLab/npm/cloud/SSH/Vault/Terraform/Kubernetes token，审计 CI/CD workflow、OIDC token 使用、异常 repo push 和云审计日志。后续应固定扩展来源与版本，降低 IDE 自动更新对生产凭据的触达。

**信息来源：**

- https://nx.dev/blog/nx-console-v18-95-0-postmortem
- https://nvd.nist.gov/vuln/detail/CVE-2026-48027
- https://www.stepsecurity.io/blog/nx-console-vs-code-extension-compromised
- https://www.cisa.gov/known-exploited-vulnerabilities-catalog

## 4. DAEMON Tools 官方下载链路被投毒：安装包分阶段投递窃密器与后门

**入选理由：** Kaspersky GReAT 披露官方站点供应链攻击，攻击影响常见 Windows 工具的可信下载路径，具备明确攻防处置价值。

**技术细节：** Kaspersky 称攻击者入侵 DAEMON Tools 官方下载站点，在 2026-04-08 至 2026-05-05 期间投放恶意版本，涉及 DAEMON Tools Lite 12.5.0.2421 至 12.5.0.2434 的相关二进制。安装后第一阶段收集主机名、MAC、进程、已安装软件、系统区域等信息；随后按目标画像投递轻量后门，可执行命令、下载文件并在内存中运行代码。厂商在 5 月 6 日后发布新版本修复。

**影响与处置：** 企业终端应排查上述版本安装记录和相关二进制哈希，隔离可疑终端，检查下载时间、命令执行、持久化和横向移动痕迹。使用软件分发平台的组织，应对“官方源下载”同样做哈希校验、签名验证和延迟发布策略。

**信息来源：**

- https://www.kaspersky.com/about/press-releases/kaspersky-identifies-ongoing-supply-chain-attack-on-official-daemon-tools-website-distributing-backdoor-malware

## 5. TanStack / Megalodon / Nx 关联供应链风险：CI/CD、扩展和工作流成为共同攻击面

**入选理由：** CISA 近期公开提醒多个供应链攻击活动，媒体与研究机构将 Nx Console、TanStack、Megalodon 等事件放在同一 DevOps 攻击面下讨论。

**技术细节：** 相关攻击集中利用开发者身份、GitHub Actions、CI/CD secrets、扩展自动更新和包仓库信任链。公开分析提到的攻击技术包括 `pull_request_target` 误用、Actions cache poisoning、运行时提取 OIDC token、恶意 workflow 注入和凭据窃取。部分信息仍来自安全厂商和媒体，组织应以各项目官方 advisory、GitHub 安全公告、CISA KEV 和 NVD 条目交叉验证。

**影响与处置：** 对所有近期受影响项目，通用处置是轮换所有可被开发机和 CI 触达的凭据，审计 workflow 文件变更、fork PR 权限、缓存边界、artifact signing 配置和包发布 token。新包或新扩展发布后，生产环境应设置观察窗口，不直接自动拉取最新版本。

**信息来源：**

- https://www.cisa.gov/news-events
- https://www.techradar.com/pro/security/cisa-warns-that-nx-console-and-github-repositories-abused-in-multiple-supply-chain-compromises-tools-across-enterprise-cloud-and-devops-environments-exploited
- https://labs.cloudsecurityalliance.org/research/csa-research-note-vscode-extension-supply-chain-breach-20260/
