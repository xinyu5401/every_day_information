# 2026-06-10 网络安全技术动态

## CISA KEV 到期窗口

### Check Point IKEv1 VPN 认证绕过 CVE-2026-50751 进入 24 小时处置窗口

**入选理由**  
CISA 已将该漏洞收入 KEV，且 `Due Date` 为 `2026-06-11`，命中今日到未来 24 小时处置窗口；同时 Check Point 与 Rapid7 均确认存在在野利用。

**技术细节**  
`CVE-2026-50751` 为 Check Point Remote Access VPN / Mobile Access / Spark Firewall 中的认证绕过，官方与 Rapid7 均指出利用前提集中在 `IKEv1`、接受旧版 Remote Access 客户端、且连接时不要求机器证书的部署。Rapid7 给出 `CVSS 9.3`，并指出漏洞根因是 IKEv1 证书校验逻辑流缺陷，攻击者可在未提供有效凭据的情况下建立 VPN 会话。公开记录显示，受影响版本涉及 `R81.20 Jumbo Hotfix Take 141 及以下`、以及 `R81.10 / R81 / R80.40` 等旧分支；供应商观察到利用最早可追溯到 `2026-05-07`，6 月初明显升温。

**影响与处置**  
这是典型边界设备高优先级事件，尤其是仍保留 IKEv1 兼容配置的公网 VPN。应立即套用官方 hotfix，禁用 IKEv1 或至少关闭 legacy client 接受路径，并核查 5 月 7 日以来异常 VPN 建链、无证书远程接入与后续横向行为。

## 1. CISA 6 月 9 日新增三项 KEV：Chrome V8、Cisco SD-WAN、Arista EOS

**入选理由**  
`2026-06-09` 的 CISA KEV 新增池一次加入 `CVE-2026-11645`、`CVE-2026-20245`、`CVE-2026-7473`，分别覆盖浏览器引擎、SD-WAN 控制面和数据中心网络设备，资产面广、优先级高。

**技术细节**  
`CVE-2026-11645` 为 Chromium `V8` 越界内存访问，Google 在 `Chrome 149.0.7827.102/.103` 稳定版更新中明确写明“已知存在在野利用”，并披露该版本共修复 `74` 项安全问题。  
`CVE-2026-20245` 为 Cisco Catalyst `SD-WAN Controller / Manager / Validator` CLI 输入校验不足，Cisco 安全公告给出 `CVSS 7.8`，攻击者在具备认证和本地落点后，可通过构造文件执行 `root` 级命令；Cisco 公告元数据还显示 `workarounds: No`。  
`CVE-2026-7473` 则影响启用了 `VXLAN / decap-groups / GRE` 等解封装配置的 Arista EOS，错误解封装会导致非预期隧道报文被转发，直接削弱网络分段。

**影响与处置**  
浏览器侧应尽快统一到 Chrome 149 新稳定版；Cisco SD-WAN 需要先按“高权限已被拿到”前提做猎杀和审计，因为这是控制面的提权链；Arista EOS 环境应优先盘点所有启用隧道解封装的交换机和 CVX 相关资产，验证补丁与上游 ACL/过滤是否到位。三项漏洞的 `Due Date` 均指向 `2026-06-23`。

## 2. Cisco SD-WAN Manager 提权漏洞 CVE-2026-20245 仍处“已被利用、无缓解”状态

**入选理由**  
虽然该漏洞已被并入上面的 KEV 新增池，但其控制面属性和“官方无 workaround”值得单独强调。

**技术细节**  
Cisco 公告显示漏洞位于 `CLI`，标题覆盖 `Catalyst SD-WAN Controller`、`Manager` 和 `Validator`。问题源于对用户输入校验不足，攻击者若已具备低门槛本地/认证权限，即可通过上传构造文件执行任意 `root` 命令。Cisco 公告元数据更新时间为 `2026-06-09T14:34:36Z`，状态为 `Updated`，而公开元数据仍显示 `workarounds: No`，意味着当前处置重点不是临时规避，而是版本修复、权限收缩与控制面审计。

**影响与处置**  
这类漏洞对传统“边界无暴露就降级处理”的思路不友好，因为很多组织会在运维跳板、MSSP 或被入侵网管段里暴露 SD-WAN 控制面。建议立即核查 `netadmin`、自动化账户、脚本上传链路以及最近 CLI 文件导入记录。
