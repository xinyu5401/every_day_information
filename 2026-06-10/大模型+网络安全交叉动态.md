# 2026-06-10 大模型+网络安全交叉动态

## 1. LiteLLM MCP 端点命令注入 CVE-2026-42271 被 CISA 收入 KEV

**入选理由**  
这是今天最典型的 `LLM + 攻击面` 交叉事件之一：对象不是通用 Web 服务，而是被广泛接在代理编排链路里的 LLM 网关；CISA 已纳入 KEV，且公开研究给出了 MCP 入口与可链式利用路径。

**技术细节**  
GitHub Advisory 显示 `CVE-2026-42271` 影响 `BerriAI/litellm`，修复版本为 `1.75.3`，漏洞等级 `CVSS 9.8`，根因是代理上的用户可控输入被直接拼接进 OS 命令。Horizon3 的公开分析指出，该缺陷可在 LiteLLM 充当 `MCP server` 时，从 `/mcp`、`/mcp/sse`、`/mcp/messages/` 等暴露端点进入；在其验证场景里，攻击者还能与 `CVE-2026-48710`（Starlette 中间件链错误）串联，绕过原本应有的鉴权逻辑。由于 LiteLLM 常被部署在统一模型网关、工具路由层或私有模型代理前端，这使问题不再是单点 RCE，而是可能演变为横向调用云模型、读取挂载密钥、劫持 Agent 工具链。

**影响与处置**  
优先级应高于普通 Python Web RCE。需要补齐的不是单一补丁，而是“代码 Agent 安全基线”：  
1. 工具入口：核查是否公开暴露 MCP 相关路径。  
2. 可访问资源：盘点 LiteLLM 进程可读写目录、模型密钥、云凭据与代理后端。  
3. 审批机制：确认工具调用是否有显式批准或 allowlist。  
4. 日志字段：至少保留 `tool_name`、`tool_args`、`caller`、`source_ip`、`session_id`、`approval_state`。  
5. 回滚机制：清理锁文件、镜像缓存、容器层和私有仓库里仍可能继续安装的旧版本。

## 2. Anthropic 发布 Claude Fable 5 / Claude Mythos 5，并同步抬高高风险生化场景防护

**入选理由**  
这是 2026-06-09 唯一确认的头部厂商旗舰级新模型发布，且并非单纯性能发布；官方把模型可用性、长上下文、价格和高风险场景安全控制绑定在同一发布里，必须按交叉领域处理。

**技术细节**  
Anthropic 官方于 `2026-06-09` 发布 `Claude Fable 5` 与 `Claude Mythos 5`。官方新闻页称 Fable 5 是当前“最智能的混合推理模型”，强调更强的代码、视觉与计算机使用能力；模型文档显示 `Fable 5 / Fable 5 Pro / Mythos 5` 默认 `200K` 上下文，并支持 `1M-token context window` 扩展。官方价格页列出 `Fable 5` 为 `$10 / 百万输入 tokens`、`$50 / 百万输出 tokens`，可用渠道包含 `Anthropic API`、`Claude.ai`、`Google Cloud Vertex AI`。安全侧，Anthropic 同步宣布高风险生化相关交互的强化保护项目 `Project Glasswing`，并在系统卡中说明其对危险生物学能力的治理定位高于前代。  

**影响与处置**  
对企业用户，这不是“换个更强模型”这么简单。若团队计划把新模型接入代码 Agent、终端 Agent 或企业 MCP 生态，应同步审查：  
1. 默认可调用工具集合是否变化。  
2. 长上下文是否扩大敏感记忆污染面。  
3. 代码/终端能力增强后，审批与审计字段是否足以追责。  
4. 高风险内容策略是否影响安全研究、漏洞验证与自动化红队流程。
