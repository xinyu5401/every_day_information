## Anthropic：Claude Managed Agents 的 webhooks、多 Agent 编排与自托管沙箱扩展到 Claude Platform on AWS

**入选理由**：Anthropic 官方 release notes 于 2026-05-29 更新 Claude Managed Agents 在 AWS 托管形态下的可用能力，涉及 Agent 执行边界、IAM 权限和自托管沙箱，是大模型 Agent 落地与安全运维的交叉更新。

**技术细节**：
- 产品：Claude Managed Agents on Claude Platform on AWS。
- 新增能力：webhooks、多 Agent 编排、自托管沙箱。
- 权限控制：官方同时更新了 AWS 侧 IAM actions，并新增 `AnthropicSelfHostedEnvironmentAccess` managed policy。
- 安全边界：自托管沙箱允许企业将工具执行环境放在自身基础设施中，而不是完全运行在 Anthropic 托管环境；这会把网络出口、凭据注入、文件系统、MCP server 访问控制与审计责任更多交给企业侧。

**影响与处置**：
- 对需要执行代码、调用内部工具或连接 MCP server 的企业 Agent，AWS IAM 与自托管沙箱提供了更清晰的隔离落点，但也要求企业补齐沙箱镜像加固、最小权限、出站访问控制、工具调用审计与 webhook 事件验签。
- 建议上线前把 Agent session、tool call、MCP server 更新、vault credential refresh、sandbox 文件访问纳入统一审计字段；对高危工具使用显式审批或策略网关。

**信息来源**：
- Anthropic Release Notes：<https://platform.claude.com/docs/en/release-notes/overview>
- Claude Platform on AWS 文档入口：<https://platform.claude.com/docs/en/build-with-claude/claude-platform-on-aws>

## OpenAI：第三方前沿模型评测指南强调 Agent harness、工具访问和攻击预算会改变网络安全能力测量

**入选理由**：OpenAI 于 2026-05-29 发布第三方评测方法论文章，核心问题不是单一模型发布，而是如何评估具备工具调用、长轨迹执行和网络安全任务能力的前沿模型。文章多处引用 cyber range、jailbreak、自定义 harness、token budget、Codex CLI 等要素，直接关系到模型网络安全能力评测的可解释性。

**技术细节**：
- OpenAI 将评测目标区分为能力诱发、系统比较、风险评估三类，并要求报告明确测试主张、任务/工具/评分方法、harness、预算、限制和有效性证据。
- 对长轨迹 Agent，harness 会影响工具使用、状态保持、错误恢复与上下文压缩；同一模型在不同 harness 下可能呈现不同能力上限。
- 网络安全场景中，文章引用 UK AISI cyber range 评测：将 token budget 从 10M 提高到 100M 可显著改变任务成功率；这意味着固定低预算结果可能只是能力下界。
- 对安全防护测试，文章强调要匹配攻击者资源模型；专家攻击者可使用自定义 harness 保存并复用绕过策略，因此仅用简单提示测试可能低估真实攻击成功率。

**影响与处置**：
- 采购或自研大模型安全评测时，不应只看单一榜单分数；必须同时记录模型版本、工具权限、Agent loop、上下文压缩、预算、拒答计入方式、污染检查和 reward hacking 复核。
- 对自动化渗透测试、漏洞复现、代码修复 Agent，应把“每次成功成本”“多次尝试成功率”“工具调用审计轨迹”纳入风险评估，而不是只给一次性通过率。

**信息来源**：
- OpenAI 官方：<https://openai.com/index/trustworthy-third-party-evaluations-foundations/>
