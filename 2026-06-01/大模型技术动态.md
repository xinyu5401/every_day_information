## xAI：`grok-build-0.1` 进入 API 公测，定位高速 Agentic Coding 模型

**入选理由**：xAI 官方新闻页显示 `grok-build-0.1` 于 2026-05-29 发布到 xAI API 公测；它与近期 OpenClaw、OpenCode、Kilo Code、Hermes Agent 等编码 Agent 生态集成有关，属于可实际调用的模型/API 更新，而非榜单热度。

**技术细节**：
- 模型/产品 ID：`grok-build-0.1`。
- 可用渠道：xAI API 公测；官方同时标注可通过 OpenRouter 与 Vercel AI Gateway 使用。
- 定位：专门面向 agentic coding，覆盖 Web 开发、调试、MCP 支持，以及一般工具调用型 Agent 场景。
- 性能/价格：官方称服务端输出速度超过 100 tokens/s；价格为 1 美元/百万输入 token、2 美元/百万输出 token。
- 适配形态：推荐放在 Grok Build CLI、Cursor、Hermes Agent、OpenClaw、Kilo Code、OpenCode 等 Agent harness 中使用。官方页面未给出上下文长度、系统卡或公开 benchmark 细项。

**影响与处置**：
- 对编码 Agent 平台而言，这是一个价格较低、速度优先的候选模型；但当前缺少公开 system/model card、上下文长度和安全边界说明，生产接入前应自行做代码执行、工具调用、MCP 参数污染与越权工具调用评测。
- 对多模型路由平台，可先把它作为“低成本快速代码子任务/工具调用子任务”候选，而不是默认替换高可靠代码审查或安全修复模型。
