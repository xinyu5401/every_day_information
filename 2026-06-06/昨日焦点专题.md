# 昨日焦点专题：MCP/Agent 供应链风险从“包污染”扩展到“工具与凭据边界”

日期：2026-06-06

## 核心判断

过去一轮监测中，开发者工具供应链事件与 MCP/Agent 安全事件出现明显交叉：npm 包污染不再只意味着安装期恶意脚本，还可能读取本地 Agent 配置、MCP server 地址、Claude/Kiro 配置文件和云端/CI 凭据；同时，NocoDB MCP `readAttachment` 漏洞说明 MCP 工具自身的资源所有权校验仍可能缺失。

## 关键事件

- Red Hat `@redhat-cloud-services` npm 污染事件中，受影响包包含 `hcc-feo-mcp`、`hcc-kessel-mcp`、`hcc-pf-mcp`，攻击链涉及 GitHub orphan commit、Actions OIDC token、npm trusted publishing 与有效 SLSA provenance。payload Miasma 读取 GitHub/npm/cloud/Kubernetes/Vault/CI 凭据，并命中 Claude/MCP/Kiro 配置路径。
- NocoDB CVE-2026-47388 表明 MCP token 持有者若知道附件路径，可调用 `readAttachment` 跨 base/workspace 读取 shared storage 中的附件，修复点是把工具参数路径回查到资源所有权记录。
- CUA-HandCrafted 论文显示，浏览器 Agent 和代码 Agent 的注入防护存在域条件差异；浏览器表面加固不能自动代表代码 Agent、CLI Agent 或 MCP 工具调用安全。

## 处置清单

- 对开发者与 CI 环境执行一次 MCP/Agent 凭据盘点：`~/.claude/mcp.json`、`~/.claude.json`、`.claude.json`、`~/.kiro/settings/mcp.json`、`.kiro/settings/mcp.json`、`.npmrc`、GitHub CLI token、云凭据和 Kubernetes service-account token。
- 对 GitHub Actions 发布链路收敛 `permissions`，尤其是 `id-token: write`；把 trusted publisher 绑定到受保护分支和具体 workflow，而不是宽泛信任整个仓库。
- 对包仓库和私有镜像清理受影响版本，并检查 lockfile、npm cache、CI 镜像层、离线构建缓存，避免撤包后继续安装。
- 对 MCP server 增加工具级授权审计：工具名、调用主体、workspace/base/project/resource ID、参数原文、外部输入来源、审批结果、返回对象大小和异常拒绝原因。
- 对代码 Agent、CLI Agent 与 IDE Agent 分离策略：默认禁用高风险工具，命令执行与文件写入需要可回放日志，密钥读取路径必须默认拒绝或脱敏。
