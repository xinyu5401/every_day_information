## CISA KEV 到期窗口

### 1. CVE-2026-45247：Mirasvit Full Page Cache Warmer 反序列化 RCE 今日进入 KEV 处置到期窗口

**入选理由**：CISA KEV `Due Date` 命中 2026-06-06；虽然 `Date Added` 是 2026-06-03，不属于只按新增日期筛选的漏洞，但符合每日到期窗口规则。

**技术细节**：漏洞影响 Mirasvit Full Page Cache Warmer for Magento 2 1.11.12 之前版本。NVD/CNA 描述为 PHP object injection：攻击者无需认证，通过 `CacheWarmer` cookie 提供构造的序列化 PHP 对象，触发不受限制的 `unserialize()`，并结合 Magento 及其依赖中的 gadget chain 实现远程代码执行。CNA 给出的 CVSS v3.1 为 9.8，CVSS v4.0 为 9.3，CWE-502。CISA KEV 字段：`Date Added: 2026-06-03`，`Due Date: 2026-06-06`，要求按厂商缓解措施处置，无法缓解则停用。

**影响与处置**：Magento 电商站点通常直接暴露互联网，且漏洞无需认证，应优先排查是否安装 `mirasvit/module-cache-warmer`，升级到 1.11.12 或更高版本；同步检查 WAF/访问日志中异常 `CacheWarmer` cookie、PHP 反序列化错误和新落地 WebShell。

**信息来源**：https://www.cisa.gov/known-exploited-vulnerabilities-catalog?field_cve=CVE-2026-45247；https://nvd.nist.gov/vuln/detail/CVE-2026-45247

## 新增在野利用与重大漏洞

### 2. CVE-2026-28318：SolarWinds Serv-U 未认证 DoS 被加入 CISA KEV

**入选理由**：CISA KEV 2026-06-05 新增，属于在野利用漏洞；Serv-U 文件传输服务常作为边界暴露资产。

**技术细节**：漏洞影响 SolarWinds Serv-U 15.5.4 之前版本以及 15.5.4 初始版本。NVD 描述为未认证攻击者可发送特制 POST 请求，利用 `Content-Encoding: deflate` 触发 Serv-U 服务崩溃，CNA CVSS v3.1 7.5（AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:H），CWE-400。CISA KEV 字段：`Date Added: 2026-06-05`，`Due Date: 2026-06-19`。

**影响与处置**：该漏洞主要影响可用性，但文件传输系统通常承载业务数据交换，持续崩溃会造成业务中断并掩护其他攻击行为。建议升级到 SolarWinds Serv-U 15.5.4 Hotfix 1 或后续版本；无法立即升级时，限制公网访问面、加固反向代理请求体/压缩处理策略，并监控异常 POST 与服务重启事件。

**信息来源**：https://www.cisa.gov/known-exploited-vulnerabilities-catalog?field_cve=CVE-2026-28318；https://nvd.nist.gov/vuln/detail/CVE-2026-28318；https://www.solarwinds.com/trust-center/security-advisories/CVE-2026-28318

### 3. CVE-2026-20245：Cisco Catalyst SD-WAN Manager 提权漏洞已被有限利用

**入选理由**：Cisco PSIRT 公告 2026-06-04 首发、2026-06-05 更新，并明确 2026 年 6 月已发现利用；SD-WAN Manager 属于关键网络管理面。

**技术细节**：漏洞位于 Cisco Catalyst SD-WAN Manager（原 SD-WAN vManage）CLI，对用户输入校验不足。具备 `netadmin` 权限的本地认证攻击者可上传构造文件，执行命令注入并提权到 root。Cisco 指出成功利用通常需要有效凭据，或先利用 CVE-2026-20182 / CVE-2026-20127 获得所需权限；CVSS v3.1 7.8（AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H），Cisco Bug ID 为 CSCwu18563。Cisco 公告显示无可用 workaround，并给出 `/var/log/scripts.log` 中可疑脚本调用的排查线索。

**影响与处置**：互联网暴露的 SD-WAN Manager 风险更高；即使漏洞要求 `netadmin` 权限，也可能作为凭据窃取后的二阶段提权。建议立即按 Cisco 固定软件矩阵升级，缩小管理面访问范围，排查 `/var/log/scripts.log`、异常 tenant list 上传、未知 CSV/脚本文件和配置变更。

**信息来源**：https://sec.cloudapps.cisco.com/security/center/content/CiscoSecurityAdvisory/cisco-sa-sdwan-privesc-4uxFrdzx

## 攻防工具与利用模块

### 4. Metasploit 新增 Apache ActiveMQ Jolokia RCE 与 Gogs rebase RCE 模块

**入选理由**：Rapid7 2026-06-05 Metasploit 周报发布 3 个新模块，其中两个是远程代码执行利用模块，具有直接攻防可操作性。

**技术细节**：`exploit/multi/http/apache_activemq_jolokia_rce` 针对 CVE-2026-34197，滥用 ActiveMQ 暴露的 Jolokia JMX-over-HTTP API `/api/jolokia/`，调用 `addNetworkConnector()` MBean，并用构造的 `brokerConfig=xbean:http://...` 让 ActiveMQ 拉取攻击者控制的 Spring XML 上下文，通过 `java.lang.ProcessBuilder` bean 实现 RCE；利用需要认证。`exploit/multi/http/gogs_rebase_rce` 针对 Gogs `<=0.14.2` 与 `<=0.15.0+dev` 的 rebase 参数注入，攻击者可通过形如 `--exec <command>` 的分支命名进入 PR merge/rebase 工作流执行命令。周报还新增 `windows/gather/windows_kernel_pointer_enum` 后渗透模块，用 `NtQuerySystemInformation` 枚举 x64 Windows kernel object pointers，并输出 CSV loot。

**影响与处置**：对暴露 Jolokia 的 ActiveMQ 与自托管 Gogs，应优先复核认证边界、禁用不必要管理接口并升级修复版本；红队和蓝队均应将新增 Metasploit 模块路径纳入检测内容，关注异常 Jolokia MBean 调用、Gogs rebase 操作和 Windows 句柄枚举行为。

**信息来源**：https://www.rapid7.com/blog/post/pt-metasploit-wrap-up-05-06-2026
