# 昨日焦点专题：Oracle PeopleSoft CVE-2026-35273 在野利用链

## 为什么是昨日焦点

过去 24 小时内，这一事件同时满足四个高优先级条件：Oracle 官方单独发布 Security Alert、Google/Mandiant 给出真实入侵链、CISA 纳入 KEV、目标系统普遍承载关键业务数据。它不是“漏洞刚公开但暂无样本”，而是已经看到勒索与窃密型对手把漏洞接进完整攻击链。

## 事件概览

- 对象：Oracle PeopleSoft PeopleTools `Updates Environment Management`
- 编号：`CVE-2026-35273`
- 风险：未认证远程代码执行
- CVSS：`9.8`
- 受影响版本：`8.61`、`8.62`
- CISA KEV：`Date Added = 2026-06-12`，`Due Date = 2026-06-15`

## 已披露的利用链重点

Mandiant 将活动归因到 `UNC6240 / ShinyHunters`。其披露的样本不是停留在“拿 shell”，而是已经展示出较完整的后续行为：

- 先在中转/暂存主机上安装 MeshCentral 与证书自动化组件，搭建外连控制面。
- 通过 `meshctrl.js` 在受害环境中执行管理命令。
- 读取 `psappsrv.cfg`、WebLogic `config.xml`、`/etc/hosts` 与挂载信息，建立 PeopleSoft 与内网主机拓扑。
- 写入并触发 `[victim]_fanout.sh`，对内部主机做 SSH 凭据喷洒与横向传播。
- 将勒索标记文件复制到 WebLogic 与 Process Scheduler 目录。
- 使用 `zstd` 打包外传数据，并向承载 ShinyHunters DLS 的 IP 建立出站 SSH 连接。

这说明该漏洞的实际风险不是单点 RCE，而是“业务系统入口 + 内网扩散跳板 + 数据泄露前置阶段”三者叠加。

## Oracle 与 Mandiant 给出的关键缓解点

- 立即应用 Oracle Security Alert 对应补丁/缓解。
- 如果短期无法完成修复，优先阻断 `/PSEMHUB/*`，特别是 `/PSEMHUB/hub`，以及 `/PSIGW/HttpListeningConnector` 的外部访问。
- 不要把 WAF 请求体检测当成充分防护，Mandiant 明确提示单靠这类规则可被绕过。
- 排查 MeshCentral 落地、异常 `meshctrl.js` 使用、SSH 横移、`README-IF-YOU-SEE-THIS-YOUVE-BEEN-HACKED.TXT` 标记、以及异常压缩外传行为。

## 对企业侧的实际含义

PeopleSoft 往往不是最显眼的公网资产，但常常连着 HR、财务、校园与后台流程系统。若组织把它视为“老业务系统，能跑即可”，这类零日会直接穿透到高价值数据域。对拥有多套 PeopleSoft、WebLogic 或历史 Oracle 中间件环境的机构，建议今天就做两类动作：

1. 资产面：确认所有对外 PeopleSoft 入口、版本、暴露路径与外联限制。
2. 痕迹面：围绕 EMHub、PSIGW、MeshCentral、SSH 横移、异常压缩和外传地址做回溯。
