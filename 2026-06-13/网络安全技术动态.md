## CISA KEV 到期窗口

### Ivanti Sentry CVE-2026-10520 进入未来 24 小时处置窗口

**入选理由**  
CISA KEV 目录命中 `Due Date` 窗口。虽然不是今天新加，但其 `Due Date` 为 2026-06-14，按本日报规则属于必须单列的到期候选。

**技术细节**  
Ivanti 将 CVE-2026-10520 描述为 Ivanti Sentry 中的 OS Command Injection，影响修复前的 `R10.5.2`、`R10.6.2`、`R10.7.1` 之前版本，可由远程未认证攻击者拿到 root 级远程代码执行。NVD 给出的 CVSS 3.1 向量为 `AV:N/AC:L/PR:N/UI:N/S:C/C:H/I:H/A:H`。Ivanti 同期还修复了 critical SQL injection 漏洞 CVE-2026-10523，但当前 KEV 到期窗口命中的是 CVE-2026-10520。

**影响与处置**  
Sentry 处在移动设备到后端企业系统之间的流量路径，通常属于边界或半边界组件。未来 24 小时内应优先确认是否仍暴露旧版本，必要时直接下线外网暴露面并升级到受支持版本。

## 1. Oracle PeopleSoft CVE-2026-35273 被 CISA 纳入 KEV，Mandiant 给出在野利用链

**入选理由**  
这是过去 24 小时内安全侧最强信号之一：Oracle 官方发布 Security Alert，Google/Mandiant 披露实际入侵活动，CISA 同步纳入 KEV。

**技术细节**  
Oracle 将 CVE-2026-35273 定义为 PeopleSoft PeopleTools `Updates Environment Management` 组件的未认证远程代码执行漏洞，影响 `8.61` 与 `8.62`，CVSS 3.1 为 `9.8`，协议面向 HTTP，且无需认证即可利用。Mandiant 将活动归因到 `UNC6240 / ShinyHunters`，观察窗口为 2026-05-27 至 2026-06-09；对手在被控主机上搭建 MeshCentral，使用 `meshctrl.js` 执行命令，读取 `psappsrv.cfg`、`config.xml` 与 `/etc/hosts` 做 PeopleSoft/内网拓扑摸排，并通过写入 `[victim]_fanout.sh` 执行 SSH 凭据喷洒和横向传播，随后压缩数据并外连到承载 Data Leak Site 的主机。CISA KEV 检索结果显示该漏洞 `Date Added` 为 `2026-06-12`，`Due Date` 为 `2026-06-15`。

**影响与处置**  
该漏洞对应互联网暴露的业务系统与关键后台组件，且已出现真实勒索/窃密链路，不应只按“常规补丁”处理。优先动作应包括：立即应用 Oracle Security Alert；若暂无法修复，至少阻断 `/PSEMHUB/*` 特别是 `/PSEMHUB/hub` 与 `/PSIGW/HttpListeningConnector` 的外部访问；同时检查 MeshCentral、异常 SSH 横移、`README-IF-YOU-SEE-THIS-YOUVE-BEEN-HACKED.TXT` 等落地痕迹。

## 2. Jenkins core 发布 2026-06-10 安全通告，`config.xml` 路径牵出反序列化与明文密钥风险

**入选理由**  
官方通告发布时间仍在日报观察窗口附近，且命中核心 CI/CD 平台，问题不只是常规 XSS，而是涉及控制器文件读取、身份冒充和配置明文回显。

**技术细节**  
Jenkins core 本次修复 6 个编号漏洞，重点是 `SECURITY-3707 / CVE-2026-53435`。在 `Jenkins 2.567` 及更早版本、`LTS 2.555.2` 及更早版本中，攻击者可借由攻击者可控的 `config.xml` 提交，让 Jenkins 反序列化任意 Jenkins core 或插件定义类型，并进一步接管 HTTP 请求处理。利用前提是具备 `Overall/Read`，并拥有账号或任一可 `POST config.xml` 的权限组合。官方明确指出该链路可导致“代表任意用户发送请求，最高到 Script Console 任意代码执行”，以及“读取 Jenkins controller 任意文件”。同批次还修复了 `CVE-2026-53442`，即 `POST config.xml` 明文 secrets 会落盘并被后续 `GET config.xml` 回显。修复版本为 `2.568` 与 `LTS 2.555.3`。

**影响与处置**  
这是典型的 CI/CD 高价值资产风险。应优先盘点是否给了低信任用户 `Item/Configure`、`View/Configure`、`Agent/Configure` 或 `Extended Read` 等权限；升级前可临时收紧 `config.xml` 相关管理权限，并审计是否出现异常 controller 文件读取与 Script Console 调用。

**信息来源**  
- Jenkins Security Advisory: https://www.jenkins.io/security/advisory/2026-06-10/
