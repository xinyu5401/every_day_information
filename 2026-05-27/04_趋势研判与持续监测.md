# 2026-05-27 趋势研判与持续监测

## 发展趋势

### 1. LLM 架构变化正在重塑安全评测

- Diffusion LLM、混合推理模型、长上下文模型和 GUI agent 会改变攻击面。
- 传统 jailbreak、拒答率和单轮问答评测不足以覆盖工具调用、长期任务、上下文裁剪和跨应用执行。
- 需要新增评测项：长上下文提示注入、工具调用链污染、记忆污染、多模态诱导、GUI 误操作和跨轮权限漂移。

### 2. Agent 安全从“内容安全”转向“执行安全”

- 模型是否说出危险内容只是低层指标；更关键的是模型是否选择了危险工具、传入了危险参数、跳过了审批或泄露了外部资源。
- MCP server、插件、浏览器自动化、代码执行器、文件系统工具和企业 SaaS API 是优先级最高的执行边界。

### 3. AI 自动化攻防会压缩漏洞处置窗口

- LLM 可把公告、补丁、日志、扫描结果和 PoC 片段快速拼接成攻击或检测流程。
- 漏洞响应需要从“CVSS + EPSS”扩展到“LLM 可利用性评分”，优先处理模型能快速复现、改写、批量扫描的漏洞。

### 4. AI 生成钓鱼进入规模化检测阶段

- 文本、页面、图片、下载诱导和多语言社工内容都可自动生成。
- SOC 应使用域名、证书、页面结构、下载物、终端行为和文本相似度联合聚类，避免只依赖单一 IOC。

## 落地动作建议

- 对内部代理平台强制落地工具调用审计：模型输入、工具选择、工具参数、执行结果、权限来源、用户确认状态、trace ID。
- 对第三方 MCP server、插件、OAuth app 和依赖包执行准入审查、版本锁定、权限最小化和定期复核。
- 为高危漏洞增加 “LLM 可利用性” 字段，纳入漏洞分级、补丁排期和红队验证。
- 将 AI 生成钓鱼纳入 SOC 标准用例，建立跨域名、证书、页面、文本与终端行为的联合聚类。
- 对 RAG 与安全知识库执行来源标记、去指令化处理、敏感数据脱敏和异常查询限速。

## 今日复核结论

- **异常热点**：OpenClaw / LLM agent 工具调用安全论文集中出现，已置顶。
- **大模型**：Hugging Face Daily Papers 与 Trending Models 显示代码、多模态、高速推理和 GUI agent 仍是高热方向。
- **网络安全**：国内官方侧未见 24 小时内同等级突发漏洞公告；CNCERT 与 CNVD AI 专区作为持续监测基线。
- **交叉领域**：自动化渗透、RAG 隐私推断、提示注入投毒、TTP 抽取和 MCP 工具治理是本窗口最值得跟踪的技术线。
