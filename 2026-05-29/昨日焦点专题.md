# 昨日焦点专题：Agent 执行安全升温（2026-05-29）

## 热点原因

过去 24 小时的交叉复核中，arXiv cs.CR recent 同时出现 OpenClaw、移动 GUI Agent 提示注入、工具型 Agent 访问控制、RAG/Multi-Agent Prompt Injection 等主题；固定关键词 OpenClaw、GUI agent security、tool attack、MCP server prompt injection、agent tracing、LLM-MAS 与 automated penetration testing 均与同一问题收敛：大模型从“生成文本”进入“读取外部内容并执行动作”后，安全边界变成运行时权限、工具参数和上下文污染控制。

## 技术核心

Agent 执行安全的核心攻击链包括四段：

1. 不可信内容进入上下文：网页、PDF、issue、聊天消息、移动 UI、OCR 文本、日志或 RAG 片段。
2. 模型把内容误当作高优先级指令：覆盖开发者指令、改变任务目标、诱导泄露上下文。
3. Agent 调用高权限工具：文件读写、网络请求、邮件发送、代码执行、部署、支付或终端控制。
4. 污染跨轮次扩散：长期记忆、多 Agent handoff、planner/executor 分离和 MCP server 间接调用放大影响。

## 已验证事实

- arXiv cs.CR recent 收录 MaskClaw、MIRAGE、AgentGuard 等与 OpenClaw、GUI Agent、工具访问控制相关论文。
- MCP 官方规范将工具安全、用户同意、授权、采样控制和本地 server 执行权限列为核心信任边界。
- OpenAI Agents SDK 文档提供 MCP 工具审批、工具过滤、输入/输出 guardrails 和 tracing，可作为工程侧落地控制点。

## 影响范围

受影响对象包括企业知识库 Agent、SOC Copilot、自动化漏洞分析工具、移动 GUI Agent、浏览器 Agent、MCP 插件生态和多 Agent 工作流。风险不是单一模型越狱，而是“外部内容注入 + 工具权限过大 + 缺少审批/审计”的组合。

## 处置建议

- 把外部内容、RAG 片段和屏幕 OCR 一律标记为不可信数据，不允许其直接改变系统策略。
- 工具调用采用 allow-list、参数校验、属性访问控制和高风险动作审批。
- MCP server 运行在最小权限沙箱内，限制文件系统、网络和环境变量访问。
- 记录 agent_id、user_id、task_id、tool_name、arguments、source_url、decision、approval、trace_id。
- 对长期记忆写入设置来源标签、可信度、过期时间和删除机制。
