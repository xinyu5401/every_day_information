# 大模型+网络安全交叉动态（2026-05-29）

## 1. MaskClaw：OpenClaw 安全、隐私与伦理风险系统化暴露

**热度判断：** arXiv cs.CR recent 收录，直接命中固定关键词 OpenClaw；该方向涉及开源 Agent 框架、工具调用和真实环境执行风险。

**技术细节：** MaskClaw 围绕 OpenClaw 类开放式 Agent 系统评估安全、隐私和伦理风险，重点关注外部工具、网页/文件内容、长期上下文和执行权限之间的耦合。风险入口包括恶意页面指令、工具返回内容污染、跨任务记忆泄露、敏感文件访问和越权外部调用。

**影响与处置：** 对使用开放 Agent 框架的团队，应将工具清单、权限范围、网络出口、文件系统访问和记忆写入纳入威胁建模。上线前至少实现工具 allow-list、敏感路径 deny-list、外部内容与系统指令隔离、执行轨迹留存和人工审批。

**信息来源：**
- https://arxiv.org/list/cs.CR/recent
- https://arxiv.org/abs/2605.28646

## 2. MIRAGE：移动 GUI Agent 面临上下文感知提示注入

**热度判断：** arXiv cs.CR recent 收录，命中 GUI agent security 与 prompt injection；移动端 GUI Agent 一旦具备点击、输入、读取屏幕和调用 App 的能力，攻击面从文本扩展到视觉界面。

**技术细节：** MIRAGE 关注移动 GUI Agent 在屏幕理解和动作规划阶段受到恶意 UI 文本、图像提示、网页内容或 App 内消息污染的问题。攻击链通常为：外部内容进入可见界面，模型把其当作高优先级任务指令，随后调用点击、输入、复制、分享或支付等动作。与普通聊天提示注入不同，GUI Agent 的污染点可隐藏在按钮、通知、网页广告、OCR 文本和聊天消息中。

**影响与处置：** 移动 Agent 应将屏幕内容标记为不可信数据，动作执行前做意图一致性校验；对支付、发信、删除、安装、权限授权等高风险动作强制二次确认。日志需记录截图哈希、UI 元素树、动作坐标、目标 App、触发文本和用户确认状态。

**信息来源：**
- https://arxiv.org/list/cs.CR/recent
- https://arxiv.org/abs/2605.28116

## 3. AgentGuard：为工具型 LLM Agent 引入属性访问控制

**热度判断：** arXiv cs.CR recent 收录，命中 tool attack 与 Agent 工具调用安全；该问题是 MCP、插件和企业 Agent 落地的共同基础风险。

**技术细节：** AgentGuard 提出属性型访问控制框架，将工具调用决策与用户身份、任务上下文、工具名称、参数、资源类型、历史动作和风险级别绑定。相比只做工具 allow-list，ABAC 能表达“谁在什么任务下能用哪个工具访问哪类资源”，并可对参数级行为做拦截，例如删除文件、外发数据、访问密钥目录或调用生产 API。

**影响与处置：** 企业 Agent 平台应把工具调用审批从 prompt 约束下沉到策略引擎：策略应在模型输出之后、工具执行之前生效，并记录 request_id、agent_id、user_id、tool_name、arguments、decision、policy_id 和审批人。对不可逆操作默认 deny 或 require approval。

**信息来源：**
- https://arxiv.org/list/cs.CR/recent
- https://arxiv.org/abs/2605.28071

## 4. RAG/Multi-Agent Prompt Injection：检索内容与 Agent 协作链污染仍是高频风险

**热度判断：** arXiv cs.CR recent 同期出现多篇提示注入、RAG 和 Agent 相关论文；该主题与企业知识库问答、SOC Copilot、自动化漏洞分析直接相关。

**技术细节：** 攻击者可把恶意指令写入网页、文档、issue、日志或工单，诱导 RAG 系统在检索后覆盖系统指令、泄露上下文、调用工具或修改输出。多 Agent 架构中，污染还会经 planner、executor、critic、memory 等角色传播，形成跨轮次上下文污染。

**影响与处置：** 检索片段必须携带来源、时间、权限和可信度标签；模型提示中应明确“检索结果是数据不是指令”。工具执行前要重新校验任务目标、用户授权和参数风险；长期记忆写入需要来源隔离和过期策略。

**信息来源：**
- https://arxiv.org/list/cs.CR/recent
- https://arxiv.org/abs/2605.28734

## 5. MCP 与 OpenAI Agents SDK：官方安全控制聚焦审批、过滤、授权和追踪

**热度判断：** MCP server prompt injection、agent tracing 是本期固定监测关键词；官方规范和 SDK 文档显示，工具调用安全正在从“提示词约束”转向协议、SDK 和运行时控制。

**技术细节：** MCP 规范将服务器视为可能访问任意数据和执行代码的高风险边界，强调用户同意、数据隐私、工具安全、采样控制和授权。OpenAI Agents SDK 提供 MCP 工具审批、静态工具过滤、tool input/output guardrails 和 tracing 等机制，可在工具执行前进行参数级拦截，并保留运行轨迹用于审计。

**影响与处置：** 建议对 MCP server 采用最小权限部署：本地 server 运行在沙箱内，HTTP transport 加认证和网段限制；一键安装必须展示完整命令和参数。Agent 平台侧应默认关闭跨 server 隐式工具调用，对 delete、write、send、pay、deploy、exec 等工具强制审批。

**信息来源：**
- https://modelcontextprotocol.io/specification/
- https://github.com/modelcontextprotocol/modelcontextprotocol
- https://github.com/openai/openai-agents-python
- https://openai.github.io/openai-agents-python/mcp/
