# 大模型+网络安全交叉动态（2026-06-09）

## 1. TanStack npm 供应链攻击：OIDC trusted publishing 被滥用，已波及 Agent/MCP/AI 开发包

- 标题：`CVE-2026-45321` TanStack 供应链攻击进入 KEV 到期窗口，攻击链明确命中 `pull_request_target`、Actions cache poisoning、`id-token: write` 与 npm trusted publishing。
- 入选理由：这是过去一段时间最具代表性的“开发者工具供应链 + 代码 Agent 生态”事件之一；今天之所以进入日报，是因为 `CISA KEV Due Date=2026-06-10` 已进入运行时点未来 24 小时处置窗口，不应因 `Date Added` 早于过去 24 小时而忽略。
- 技术细节：
  - TanStack 官方复盘确认，攻击者在 `2026-05-11 19:20-19:26 UTC` 间发布 `42` 个 `@tanstack/*` 包的 `84` 个恶意版本。
  - 核心攻击链为三段拼接：
    1. `pull_request_target` workflow 在 base 仓库信任上下文中 checkout fork 的 PR merge ref，并执行 fork 控制代码。
    2. 恶意代码把 payload 写入 `pnpm` store，通过 `actions/cache` 的 post-job save 污染主仓缓存。
    3. 发布 workflow 配置了 `id-token: write` 用于 npm OIDC trusted publishing；攻击者在 runner 内存中提取 OIDC token，绕过正常 publish step，直接向 `registry.npmjs.org` 发包。
  - TanStack 官方 IOC 给出关键指纹：恶意 `optionalDependencies` 指向 `github:tanstack/router#79ac49eedf774dd4b0cfa308722bc463cfe5885c`，恶意文件为 `router_init.js`，缓存键为 `Linux-pnpm-store-6f9233a50def742c09fde54f56553d6b449a535adf87d4083690539f49ae4da11`。
  - NVD 在该 CVE 的已知受影响软件配置中，已列出多个 AI/Agent 相关组件，如 `agentwork-cli`、`dirigible-ai/sdk`、`guardrails_ai`、`nextmove-mcp` 等，说明该事件不只是前端生态问题，已经触及 MCP/Agent/AI 工具链。
- 影响与处置：
  - 工具入口：`npm install` / `pnpm install` / `yarn install`，以及一切会消费受影响包、锁文件、缓存或私有镜像的开发机和 CI runner。
  - 可访问资源：本地文件、SSH 密钥、云元数据、`~/.npmrc`、`gh` token、Vault、Kubernetes service-account token；若安装发生在 CI/CD 上，还包括 OIDC token、仓库 secrets 和发布权限。
  - 审批机制缺口：单靠 provenance 或 trusted publishing 不能阻止“合法 workflow 上的恶意代码路径”发包；`id-token: write` 一旦存在，就要审计 token 生成点与实际发布调用点。
  - 建议动作：
    - 立即搜索锁文件、制品缓存、私有镜像、npm cache 是否包含恶意版本或 IOC commit。
    - 审计所有 `pull_request_target` workflow 是否 checkout 了 fork 代码，是否复用与生产发布同域的 cache key。
    - 梳理所有启用 OIDC trusted publishing 的 workflow，逐项核对 `permissions`、`id-token: write`、发布来源、attestation/provenance 与异常 workflow run。
    - 对命中 Agent/MCP/AI 包的环境，加查外部内容到工具参数的传播路径、默认可调用工具、凭据挂载与审批日志字段。
- 信息来源：

  - https://tanstack.com/blog/npm-supply-chain-compromise-postmortem

## 2. Nx Console 恶意扩展：TanStack 上游投毒已扩散到 VS Code / Cursor 终端

- 标题：`CVE-2026-48027` Nx Console 恶意扩展进入 KEV 到期窗口，受影响面覆盖 `VS Code` 及其 fork，如 `Cursor`。
- 入选理由：这不是普通扩展下架事件，而是典型的“上游 npm 供应链攻击 -> 开发者 `gh` 凭据泄露 -> IDE 扩展投毒 -> 终端与云凭据外传”的跨层级攻击链；同时命中今日 `CISA KEV` 到期窗口。
- 技术细节：
  - Nx 复盘确认，恶意 `Nx Console 18.95.0` 在 `2026-05-18 12:30-13:09 UTC` 期间曾在 Visual Studio Marketplace 与 Open VSX 可下载。
  - 上游起点并非 Nx 自身开发流程，而是一名核心贡献者在外部仓库执行 `pnpm install` 时安装了 TanStack 恶意包 `@tanstack/zod-adapter@1.166.15`；该仓库虽设置了 `minimum-release-age=10080`，但 `packageManager` 固定到 `pnpm 10.14`，该版本对该配置静默忽略，保护未生效。
  - 恶意扩展激活后会落地持久化文件与后门，如 `~/.local/share/kitty/cat.py`、`/var/tmp/.gh_update_state`、Windows `%USERPROFILE%\\.bun\\bin\\bun.exe`，并窃取 `gh` token、`.npmrc`、SSH、云凭据、Vault、1Password CLI 会话内容，通过 HTTPS 与 DNS 通道外传。
  - Nx 明确指出：`VSCode and its forks (e.g. Cursor) are open to the attack`；当前安全版本自 `18.100.0` 起。
- 影响与处置：
  - 工具入口：VS Code/Cursor 扩展自动更新与扩展激活。
  - 可访问资源：本地源码、shell 环境变量、云 CLI 会话、开发者 OAuth token、SSH 私钥、`.env` 文件；若用户在 IDE 中接入代码 Agent/MCP server，则其挂载凭据与工具配置同样可能暴露。
  - 命令执行权限：扩展代码在本地用户上下文执行，可读写用户目录并发起外连；若终端内已登录 `gh`、`aws`、`gcloud`、`op`，风险显著放大。
  - 审批与审计建议：
    - 扩展发布链路引入双人审批、环境保护规则与异常发布告警。
    - 开发者机禁止长期落地 `gh` CLI token，改为 secret manager 包裹调用。
    - 记录扩展版本、发布时间、auto-update 事件、激活时间、本地 IOC 命中、外联域名与凭据轮换结果。
    - 对 IDE Agent/MCP 场景，补齐默认工具清单、审批按钮、命令执行日志与回滚机制。
- 信息来源：

  - https://nx.dev/blog/nx-console-v18-95-0-postmortem
