## 核心动态

### 1. MobileGym 将 GUI agent 评测推向可复现并行仿真

- 信息来源：
  - Hugging Face Daily Papers: https://huggingface.co/papers
  - MobileGym 论文页: https://huggingface.co/papers/2605.26114
  - arXiv: https://arxiv.org/abs/2605.26114
- 技术要点：MobileGym 提供浏览器托管的移动端仿真环境，以结构化 JSON 表示完整状态，支持 fork、比较、确定性判分与并行 rollout。其 MobileGym-Bench 覆盖 28 个应用、416 个参数化任务模板，并使用 AnswerSheet 协议减少自由文本判分误差。
- 安全研判：GUI agent 一旦能稳定模拟移动 App 操作，企业需要把权限确认、敏感页面读取、跨 App 数据搬运、自动点击与支付/审批动作纳入审计。该论文不是漏洞披露，但显著提高了 GUI agent 行为评测与红队复现能力。
- 状态：动态榜单待核实。

### 2. Trajel 将多 Agent 失败检测从“最终答案”推进到轨迹级审计

- 信息来源：
  - Hugging Face 论文页: https://huggingface.co/papers/2605.24219
  - arXiv: https://arxiv.org/abs/2605.24219
- 技术要点：Trajel 针对多 Agent 工业工作流的 Thought-Action-Observation 中间步骤，构建轨迹级幻觉审计框架和五类错误 taxonomy，包括事实、指代、逻辑、流程和范围错误。
- 安全研判：这类轨迹审计可直接迁移到安全运营 Agent、自动化渗透测试 Agent 和代码修复 Agent。若只审查最终结论，可能漏掉中间工具参数污染、错误资产选择、越权查询和错误修复链路。
- 状态：动态榜单待核实。

### 3. VitaBench 2.0 与 Persistent AI Agents 共同指向长期上下文污染风险

- 信息来源：
  - VitaBench 2.0: https://huggingface.co/papers/2605.27141
  - Persistent AI Agents: https://arxiv.org/abs/2605.26870
- 技术要点：VitaBench 2.0 评测个性化与主动式 Agent 在长期交互中持续抽取、更新和使用用户偏好的能力；Persistent AI Agents 则以真实研究环境为案例，记录长期记忆、工具、仓库、计划任务与治理规则的运行数据。
- 安全研判：长期记忆会把一次性的恶意内容、错误偏好或污染指令固化为后续决策上下文。企业 Agent 平台应把长期记忆写入、检索命中、上下文拼接和遗忘策略作为审计对象，而不只记录用户输入和模型输出。
- 状态：待核实影响范围。

## 规范与工程约束

- MCP 规范强调资源 URI 校验、敏感资源访问控制、工具安全、用户同意与本地 MCP server 沙箱。对一键配置本地 server 的客户端，应展示即将执行的命令并要求显式确认。
- OpenAI Agents SDK 文档显示 guardrails、tool input/output guardrails、handoff input filter、sessions 与 tracing 是 Agent 工程的核心能力；tracing 应覆盖 LLM 生成、工具调用、handoff、guardrail 与自定义事件。
- 企业落地建议：记录模型输入、工具选择、工具参数、工具结果、权限来源、用户确认状态、handoff 目标、长期记忆命中与写入事件。对高权限工具采用默认拒绝、人工确认、沙箱执行和出网限制。
