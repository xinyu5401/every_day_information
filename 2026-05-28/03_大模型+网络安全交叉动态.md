## 1. D^2-Monitor：面向 Diffusion LLM 的动态安全监测

- 信息来源：
  - Hugging Face 论文页: https://huggingface.co/papers/2605.25893
  - arXiv: https://arxiv.org/abs/2605.25893
- 大模型技术点：Diffusion LLM 通过多步去噪生成文本，暴露出自回归模型不具备的中间隐状态轨迹。D^2-Monitor 利用“安全犹豫”信号判断轻量 probe 是否可能失败，并在高风险样本上路由到更强 probe。
- 安全技术点：该方法面向 always-on 安全分类、毒性检测和 moderation 资源调度，评测覆盖 WildGuardMix、ToxicChat 与 OpenAI-Moderation 等数据集。
- 趋势判断：diffusion LLM 的生成轨迹给安全监测提供了新信号，也意味着现有只看最终输出的安全评测需要更新。
- 状态：动态榜单待核实。

## 2. Agent 执行安全：GUI、长期记忆、轨迹审计成为交叉热点

- 信息来源：
  - Hugging Face Daily Papers: https://huggingface.co/papers
  - MobileGym: https://huggingface.co/papers/2605.26114
  - Trajel: https://huggingface.co/papers/2605.24219
  - VitaBench 2.0: https://huggingface.co/papers/2605.27141
  - Persistent AI Agents: https://arxiv.org/abs/2605.26870
- 大模型技术点：GUI agent、长期记忆、多 Agent 轨迹、Agent benchmark 与强化学习 rollout 正在从研究走向工程工具链。
- 安全技术点：这些能力扩大了提示注入、工具参数污染、长期上下文污染、错误 handoff、未授权工具调用和敏感数据外传的攻击面。
- 趋势判断：Agent 安全控制的最小闭环应包含工具调用审计、权限来源、人工确认、轨迹级检测、长期记忆治理和回放复现能力。
- 状态：待核实影响范围。

## 3. MCP 与 Agents SDK 文档显示工具调用安全进入工程规范层

- 信息来源：
  - MCP 规范: https://modelcontextprotocol.io/specification/2025-11-25
  - MCP Security Best Practices: https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices
  - OpenAI Agents SDK: https://github.com/openai/openai-agents-python
- 大模型技术点：MCP 将 LLM 应用与外部资源、工具和工作流连接；Agents SDK 提供 tools、handoffs、guardrails、sessions 与 tracing 等 Agent 原语。
- 安全技术点：MCP server 本地执行、资源 URI、OAuth/授权绑定、工具输入输出、handoff 历史、trace 数据都成为安全边界。工具输入/输出 guardrail、handoff 过滤和 tracing 是降低 prompt injection 与越权调用风险的关键控制点。
- 趋势判断：MCP server 与插件应按高权限执行面管理，不能只按“模型上下文扩展”管理。

## 4. AI 安全技能库上榜，安全自动化开始插件化

- 信息来源：
  - GitHub Trending: https://github.com/trending
  - Anthropic-Cybersecurity-Skills: https://github.com/mukul975/Anthropic-Cybersecurity-Skills
- 大模型技术点：安全知识被封装为 Agent 可发现、可调用的结构化 skill，适配 Claude Code、GitHub Copilot、OpenAI Codex CLI、Cursor、Gemini CLI 等平台。
- 安全技术点：技能库映射 ATT&CK、ATLAS、D3FEND、NIST CSF、NIST AI RMF，利于标准化安全作业；同时也引入技能供应链风险，包括恶意技能、过时步骤、危险命令和隐式外传。
- 趋势判断：企业采用安全 Agent skill 前应建立准入审查、版本锁定、危险动作提示、执行日志与回滚机制。
- 状态：动态榜单待核实。
