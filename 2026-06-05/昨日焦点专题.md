## 2. Red Hat npm/Miasma：trusted publishing 供应链攻击进入 Agent/MCP 风险面

Red Hat `@redhat-cloud-services` npm 包污染事件不只是普通恶意包问题：多个分析指向 `preinstall` 执行、CI/CD 环境凭据窃取、GitHub Actions OIDC trusted publishing 链路滥用，以及 MCP 相关包名命中。对代码 Agent 和 MCP server 场景，`npm install` 常由自动化工具触发，凭据又常挂载在 runner、开发容器或 IDE 环境中，攻击链的可见性比传统人工安装更差。

短期建议：排查受影响包版本、lockfile、私有镜像和缓存；轮换云凭据、Vault、Kubernetes、GitHub 和 npm token；收紧 GitHub Actions `permissions`，尤其是 `id-token: write`；对 Agent 自动执行依赖安装引入审批、隔离网络和工具调用日志。
