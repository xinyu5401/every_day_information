## PAN-OS GlobalProtect 认证绕过 CVE-2026-0257：Rapid7 补充在野利用细节与 IOC

**入选理由**  
Rapid7 对 `CVE-2026-0257` 的在野利用观察在 2026-06-03 仍有更新，补充了成功建立 VPN 隧道后的请求路径与 IOC；该漏洞位于边界 VPN 入口，优先级高于普通中危评分。

**技术细节**  
该漏洞影响启用了 `authentication override` 的 `PAN-OS` / `Prisma Access` GlobalProtect 门户或网关；若认证覆盖 cookie 的加解密证书与其他功能复用，攻击者可从公开证书链中取公钥，伪造认证覆盖 cookie，从而在未持有凭据的情况下通过网关建立 VPN 会话。Rapid7 表示其最早在 `2026-05-17` 观察到成功利用，`2026-05-21` 出现第二波；在成功建立隧道的样本中，看到了对 `/ssl-vpn/hipreport.esp` 与 `/ssl-vpn/getconfig.esp` 的访问。Rapid7 还指出 Palo Alto 最初给出的 `CVSSv4 4.7` 已在 2026-05-29 调整到 `7.8`，并要求按最高优先级修补。

**影响与处置**  
这是典型的互联网暴露边界设备风险，受影响资产即使原始评分不高，也应按“已观测利用的身份绕过”处理。优先动作是升级到厂商修复版本；若无法立即升级，应关闭 `authentication override`，或为该功能单独生成专用证书，避免与 HTTPS 服务等功能复用。同时应在 GlobalProtect 日志中排查异常 cookie 登录、可疑主机名与异常 POST 到上述 URI 的痕迹。

- https://security.paloaltonetworks.com/CVE-2026-0257

## Gogs “Rebase before merging” 参数注入可致认证后 RCE，Rapid7 指出默认配置放大可利用面

**入选理由**  
Rapid7 在 2026-05-28 披露该漏洞时明确给出完整利用链、受影响版本与 Metasploit 模块，且厂商当时尚未发布补丁；这类自托管 Git 服务 RCE 兼具横向渗透与供应链篡改价值。

**技术细节**  
漏洞位于 Gogs 在处理 `Rebase before merging` 合并方式时对 `git rebase` 参数的构造。攻击者可先推送恶意分支名，例如 `--exec=<payload>`，再在合并阶段把该分支名注入 `git rebase` 的参数位置，使 Git 把其当作 `--exec` 选项执行。Rapid7 给出的根因是 `internal/database/pull.go` 中传参时缺少 `--` 终止符，导致来自 `pr.BaseBranch` 的值被解释为选项。其确认 `Gogs 0.14.2` 与 `0.15.0+dev` 受影响，且此前支持该合并样式的版本大概率同样受影响；Rapid7 给出 `CVSSv4 9.4`，并提供 Metasploit 模块。由于 `DISABLE_REGISTRATION=false` 与 `MAX_CREATION_LIMIT=-1` 是默认值，很多默认实例会把门槛降到“攻击者仅需自注册账户并创建仓库”。

**影响与处置**  
攻击成功后可获得 Gogs 服务进程权限，读取同实例全部私有仓库、密码哈希、API token、SSH key 与 2FA secret，并直接对托管代码做文件级篡改，绕过常规审计链。短期缓解应禁用 `Rebase before merging`、关闭开放注册、限制普通用户建仓，并把 Gogs 进程与仓库存储做最小权限隔离；若业务依赖该功能，应优先评估替换或前置访问控制。
