# 大模型+网络安全交叉动态

## Anthropic 复盘一年 AI 赋能攻击样本：高风险行为正向横向移动与权限提升迁移

**入选理由**  
Anthropic 于 2026-06-03 发布官方分析，样本来自 `2025-03` 到 `2026-03` 间因恶意网络活动被封禁的 `832` 个账户，并映射到 MITRE ATT&CK；这类带真实平台治理数据的报告比泛泛“AI 攻防趋势”更有参考价值。

**技术细节**  
Anthropic 表示，`67.3%` 的样本曾用 AI 协助编写恶意软件，`6.5%` 的样本已把 AI 用到横向移动等更复杂的后渗透阶段。其风险评分结果显示，中高风险及以上样本占比在两个半年窗口内从 `33%` 升至 `56%`。更关键的是，官方认为传统以“技术数目”或“使用界面类型”区分威胁等级的方法正在失效，因为较低技能攻击者也可借助模型执行更多高复杂度 ATT&CK 技术；真正更稳定的区分信号，转向了攻击者是否构建了能让模型串联多阶段攻击、并最小化人工介入的自动化脚手架。

**影响与处置**  
对模型厂商、SOC 与红队来说，这意味着仅靠“阻断恶意提示词”已经不够，必须把检测重心前移到 agent orchestration、工具串联、连续决策和数据外流链路。做 AI 安全治理时，建议把“模型是否被用于后渗透阶段”“是否形成多步自动化 scaffold”纳入高风险判定，而不是只看单轮生成内容。

**信息来源**  
- https://www.anthropic.com/news/AI-enabled-cyber-threats-mitre-attack
- https://red.anthropic.com/2026/attack-navigator/

## 针对专有 Agent 的渗透测试论文给出实证：提示注入仍可穿透工具白名单并升级到 RCE

**入选理由**  
`Lessons from Penetration Tests on Large-Scale Agent Systems` 虽发表于 2026-05-26，但仍是最近少见的、直接针对专有 Agent 产品的渗透测试实证研究，且与代码 Agent、GitHub 助手和工具调用安全高度相关。

**技术细节**  
论文复盘了 2025 年对两个专有 Agent 系统的渗透测试。其一是面向 GitHub Issue 修复的双阶段助手：先定位，再生成补丁，并配有文件浏览工具与受限 terminal 工具。作者指出，攻击面不只在用户 prompt，也在 issue 评论、仓库内容和后续评审链；即便终端命令做了 allowlist、禁旗标与重定向限制，攻击者仍可借助提示注入驱动非预期工具调用，进一步达成仓库完整性破坏、数据外流，甚至远端代码执行。论文特别批评了“依赖用户人工复核”和“模式匹配过滤”这类弱缓解策略。

**影响与处置**  
对代码 Agent、IDE Agent、CLI Agent 与 MCP server 生态，这篇论文的现实意义在于：工具白名单不等于安全边界，真正关键的是不可信外部内容到工具参数的传播控制、执行前审批、最小资源暴露、可审计日志和失败回滚。若团队正在上线自动修复或自动提交 PR 的 Agent，应把 issue/comment/repo 三类输入全部视作不可信源。

**信息来源**  
- https://arxiv.org/html/2605.27042v1
- https://arxiv.org/abs/2605.27042

## “Provably Secure Agent Guardrail” 尝试把 Agent 安全控制从语义对齐迁移到形式化约束

**入选理由**  
该论文近期在 Agent 安全检索中持续被引用，原因不是其结论已可直接落地，而是它代表了一条与经验型 prompt guardrail 完全不同的技术路线：在执行前把意图先形式化，再做可验证约束。

**技术细节**  
论文主张放弃对自然语言语义本身的信任，让 Agent 在执行物理操作前，先把意图无损转译为一阶逻辑约束，再用形式化验证层决定是否放行。作者宣称其 `Proof-Constrained Action` / `ePCA` 思路在所评估的二维动态对抗场景里实现了 `zero attack success rate` 与 `zero false positive rate`，并把威胁模型聚焦到时间拆分与跨层权限提升攻击。但论文也明确承认三个核心限制：意图到形式化约束的翻译本身仍脆弱；复杂开放环境难以覆盖全部状态迁移；最终安全边界仍依赖人工编码的领域公理。

**影响与处置**  
这条路线短期更像“高价值受限工具”的研究方向，而不是通用 Agent 的现成防线。但它给 MCP/工具调用安全一个明确信号：如果工具参数和允许动作能先收敛到结构化、可证明的状态机，再谈审批与白名单，安全收益会比继续堆叠语义分类器更稳。

**信息来源**  
- https://arxiv.org/html/2605.29251
- https://arxiv.org/abs/2605.29251
