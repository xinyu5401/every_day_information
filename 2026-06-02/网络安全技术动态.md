# 网络安全技术动态日报（2026-06-02）

## CISA KEV 新增 Oracle WebLogic CVE-2024-21182，在野利用处置窗口至 2026-06-04

**入选理由**：CISA Known Exploited Vulnerabilities Catalog 在 2026-06-01 新增 CVE-2024-21182。该条目命中过去 24 小时 `Date Added` 条件，且对象为常见互联网暴露中间件。

**技术细节**：CISA 描述该漏洞影响 Oracle WebLogic Server，未认证攻击者可通过网络访问 T3、IIOP 触发，成功攻击可导致对关键数据的未授权访问，或获得 WebLogic Server 可访问数据的完全访问能力。CISA 记录 `Date Added=2026-06-01`，`Due Date=2026-06-04`，`Known Ransomware Campaign Use=Unknown`。CISA 引用 Oracle 2024 年 7 月 CPU 与 NVD 记录作为厂商/CVE 来源。

**影响与处置**：优先排查互联网暴露、跨网段可达或承载关键业务的 WebLogic；限制 T3/IIOP 暴露面，按 Oracle CPU 建议修复或应用缓解，无法修复的实例应下线或隔离。资产管理侧应按 CISA due date 建立短周期闭环，而不是只按漏洞披露时间排序。

**信息来源**：

- https://www.oracle.com/security-alerts/cpujul2024.html
- https://nvd.nist.gov/vuln/detail/CVE-2024-21182

## CISA KEV 2026-06-03 到期窗口集中命中旧版 Windows、Adobe Reader、IE 与 Microsoft Defender 漏洞

**入选理由**：CISA KEV 中多条 `Date Added=2026-05-20` 的在野利用漏洞 `Due Date=2026-06-03`，命中“当天或未来 24 小时处置窗口”规则。虽然不是 6 月 2 日新增，但处置优先级应上调。

**技术细节**：本次到期窗口包括：CVE-2008-4250（Microsoft Windows Server Service RPC 路径规范化缓冲区溢出，MS08-067）、CVE-2009-1537（Microsoft DirectX DirectShow QuickTime Movie Parser Filter 空字节覆盖）、CVE-2009-3459（Adobe Acrobat/Reader 堆缓冲区溢出）、CVE-2010-0249 与 CVE-2010-0806（Internet Explorer use-after-free，EoL/EoS 风险）、CVE-2026-41091（Microsoft Defender 本地提权，CWE-59）和 CVE-2026-45498（Microsoft Defender 拒绝服务）。CISA 对上述条目的勒索软件使用情况均标记为 Unknown。

**影响与处置**：这些条目横跨遗留桌面软件、浏览器组件和终端防护组件。建议把仍运行旧版 Windows/IE/Adobe Reader 的隔离网、工控网、VDI 镜像和历史业务终端列为排查重点；对 Defender 相关条目，应确认安全产品版本、补丁状态与本地低权限攻击面。IE 与旧 Adobe Reader 如已失去维护，应优先替换或强隔离。

## Rapid7 披露 HP Poly VVX/Trio VoIP 电话 CVE-2026-0826，未认证 SIP 请求可触发 root RCE

**入选理由**：Rapid7 于 2026-06-01 发布技术分析，披露 HP Poly VVX 与 Trio VoIP 设备的关键未认证栈溢出漏洞，并已开发 Metasploit 演示模块。该漏洞影响企业语音基础设施，且具备远程未认证利用条件。

**技术细节**：CVE-2026-0826 影响 VVX 150/250/350/450 以及 Trio 8300/8500/8800。Rapid7 在 VVX 450、固件 6.4.7.4477 上验证：设备的 `polyapp` 进程解析 UDP 5060 SIP 请求中的 SDP ICE candidate 属性时，`ParseICECandidate` 将外部字符串复制进 256 字节栈缓冲区且缺少长度检查，造成 CWE-121 栈溢出。漏洞 CVSS v4 为 9.2。利用链可控制程序计数器，绕过 NX；由于共享库加载地址固定，ROP 链可在无信息泄露情况下构造，最终获得 root 权限。

**影响与处置**：VoIP 电话通常缺乏 EDR 监控，且位于会议室、办公室、服务台等高价值音频环境。Rapid7 给出的修复版本为 VVX UCS 6.4.8、Trio 8300 UCS 8.1.7、Trio 8500/8800 UCS 7.2.8。建议立即升级固件，限制 UDP/5060 暴露，隔离语音 VLAN，审计异常 SIP 流量和设备配置变更；高管会议室、交易台、医疗站点等位置应优先处置。

- https://www.rapid7.com/blog/post/ve-cve-2026-0826-how-an-old-bug-can-feed-ai-powered-impersonation

## Red Hat 相关 npm 包遭 Miasma/Mini Shai-Hulud 变种污染，GitHub Actions OIDC trusted publishing 被滥用

**入选理由**：Aikido Security 于 2026-06-01 披露，`@redhat-cloud-services` 作用域下 32 个包、96 个版本被植入凭据窃取蠕虫。该事件命中开发者工具供应链固定监测项：package registry compromise、GitHub Actions、OIDC token、developer secrets。

**技术细节**：Aikido 报告称，攻击者疑似先控制 Red Hat 员工 GitHub 账号，然后向多个仓库推送孤立提交，加入 `ci.yaml` 与 `_index.js`。工作流授予 `id-token: write` 权限，安装 Bun 后运行脚本，通过 GitHub Actions OIDC 获取短期 token，再调用 npm trusted publishing 端点发布后门版本。受影响包在 `package.json` 中声明 `preinstall`，安装时自动执行 `node index.js`。载荷扫取 GitHub Actions secrets、`GITHUB_TOKEN`、`ACTIONS_RUNTIME_TOKEN`、AWS/GCP/Azure 凭据、Vault token、Kubernetes token/kubeconfig、npm/PyPI token、SSH 私钥、Docker 凭据、GPG key 与 `.env` 文件。受影响列表中还包含 `@redhat-cloud-services/hcc-feo-mcp`、`hcc-kessel-mcp`、`hcc-pf-mcp` 等 MCP 相关包。

**影响与处置**：凡 2026-06-01 后安装受影响版本的开发者机器、CI runner 或构建镜像，应按凭据泄露处理：撤回相关包版本，清理 lockfile 与缓存，轮换云密钥、npm/PyPI 发布 token、SSH key、GitHub Actions secrets 和 OIDC 信任策略；检查仓库是否存在孤立提交、异常 workflow、`id-token: write` 权限滥用和异常 trusted publishing 记录。对 MCP 包使用方，还需确认 MCP server 是否被污染并重新构建可信产物。

**信息来源**：
- https://www.aikido.dev/blog/red-hat-npm-packages-compromised-credential-stealing-worm

## CNCERT 通报家用路由器 DNS 被恶意篡改，单日影响境内独立 IP 最高超过 70 万

**入选理由**：CNCERT 于 2026-06-01 发布风险提示，指出部分家庭用户连接家用路由器 Wi-Fi 后访问正常网站会异常跳转至色情、赌博等非法页面。该事件有明确攻击入口、影响规模和处置建议。

**技术细节**：CNCERT 研判事件主要由家用路由器等家庭网络设备 DNS 服务器配置被篡改导致。现场取证显示，受影响路由器后台管理口令强度不足，攻击者疑似先感染家庭内网设备，再利用弱口令进入路由器管理页面修改 DNS 配置。截至通报时，CNCERT 已监测到数十个提供恶意解析服务的 DNS 服务器和异常跳转 Web 服务器，单日恶意解析次数超过数亿次，单日影响境内独立 IP 最高超过 70 余万个。

**影响与处置**：家庭用户应检查路由器 WAN/LAN/DHCP DNS 配置，恢复为运营商自动下发 DNS 或可信公共 DNS，修改后台弱口令，关闭非必要远程管理、UPnP 与端口映射，升级固件；出现多终端同 Wi-Fi 异常跳转时，应断网排查并必要时恢复出厂设置。路由器厂商应强化后台登录保护、弱口令提示、异常 DNS 告警和固件更新。

**信息来源**：
- https://www.cert.org.cn/publish/main/11/2026/20260601145618109326016/20260601145618109326016_.html
