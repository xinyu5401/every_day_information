## 1. arXiv：LLM 安全研究继续从单点提示攻击转向系统级 agent 风险

- **动态**：arXiv cs.CR / cs.CL recent 仍集中出现 LLM agent、安全评测、提示绕过、数据投毒、长上下文与多工具环境相关研究。
- **技术要点**：
  - 传统 jailbreak 评测不足以覆盖工具调用、上下文压缩、记忆污染和外部环境反馈。
  - agent 论文越来越强调 action trace、tool result、environment state 对模型行为的影响。
  - 长上下文模型的安全边界需要重新评估：安全过滤器看到的文本、模型实际关注的片段、工具参数最终落地的内容可能不一致。
- **影响**：内部大模型安全评测不应只测 prompt refusal，应加入工具权限、执行结果污染、长上下文截断、记忆写入和跨工具链路。
- **来源**：arXiv cs.CR recent <https://arxiv.org/list/cs.CR/recent>；arXiv cs.CL recent <https://arxiv.org/list/cs.CL/recent>。

## 2. Hugging Face：模型和轨迹数据资产继续推动 agent 化训练

- **动态**：Hugging Face Trending Models / Datasets 中，reasoning、agent traces、instruction 数据和工具调用样本仍是活跃类别（具体项目排名需持续核验）。
- **技术要点**：
  - 轨迹数据可提升模型分解任务和使用工具的能力。
  - 数据集中若包含敏感 prompt、内部工具返回、真实用户任务或未脱敏日志，可能造成隐私与复现风险。
  - 被污染或恶意构造的轨迹样本可能把不安全的工具选择偏好固化进微调模型。
- **影响**：使用公开 agent traces 做训练或评测前，应执行 PII 检测、命令片段检测、外链检测、恶意意图标签抽样和许可证复核。
- **来源**：Hugging Face Models Trending <https://huggingface.co/models?sort=trending>；Datasets Trending <https://huggingface.co/datasets?sort=trending>。

## 3. GitHub：AI agent 基础设施项目继续高频出现

- **动态**：GitHub 日榜中持续出现 AI agent、MCP、代码图谱、浏览器自动化、Claude Code / Codex / Cursor / OpenCode 辅助工具等项目。
- **技术要点**：
  - 代码图谱类项目使模型能跨文件理解仓库，安全风险在于扩大敏感代码暴露面。
  - 浏览器自动化 / DevTools MCP 使模型能读取页面、点击、提交表单，安全风险在于会话劫持、CSRF 类误操作、敏感页面截图与表单泄露。
  - 本地 agent 工具如果默认授予 shell、文件系统、网络或浏览器权限，应按高危插件处理。
- **影响**：开发团队在引入此类项目时，应默认启用最小权限、只读模式、工作区隔离和工具调用审批。
