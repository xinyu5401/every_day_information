### 技术拆解

- **攻击面扩大**：MCP server、browser automation、代码知识图谱、本地文件工具和云 API 工具被统一暴露给模型后，攻击面不再只是 prompt，而是包含工具描述、参数 schema、权限来源、返回内容和执行环境。
- **审计字段明确化**：OpenAI / Realtime / MCP 相关文档显示，安全审计至少应覆盖 `tool_choice`、`tools`、`tool_calls`、`arguments`、`output`、`server_label`、`allowed_tools`、`mcp_list_tools`、`mcp_approval_request`、`mcp_approval_response`、`mcp_call`、`approval_request_id`、`status`、`error`。
- **供应链风险上升**：安全技能库和 agent 插件常由 Markdown、YAML、脚本、提示模板和外链组成。恶意维护者可通过提示注入、过宽权限模板、伪造 ATT&CK / D3FEND 映射、危险 shell 片段或依赖投毒影响下游 agent。
- **防守价值同步存在**：结构化技能库可加速 SOC、DFIR、云安全审计和漏洞管理，但前提是准入审查、签名、版本锁定、沙箱执行和结果复核机制到位。

### 趋势判断

- **短期**：MCP server 与 AI 安全技能项目会继续在 GitHub / Hugging Face 上快速迭代，质量参差不齐。
- **中期**：企业内部 agent 平台会从“能不能调用工具”转向“如何审计每一次工具调用”。
- **长期**：AI 安全评测将从 jailbreak 成功率扩展到“工具权限、长上下文污染、轨迹数据污染、审批绕过、跨 server 调用链”。

### 建议

- 对所有第三方 MCP server、插件、技能库实施准入审查：来源、许可证、维护者、依赖、脚本、网络访问、文件访问、凭据访问。
- 在代理平台日志中强制记录工具调用全链路字段：模型输入、候选工具、实际工具、参数、返回值、权限来源、审批请求、审批结论、执行环境、调用链路 ID。
- 对安全技能库做攻防双重评估：既评估 SOC 增效，也评估是否能让低权限用户生成可执行利用链。
- 将“GitHub Trending AI agent / MCP / security skill”列为每日雷达项，发现星标异常增长或高权限工具项目上榜时单独复核。
