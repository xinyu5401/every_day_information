# 2026-05-26 大模型+网络安全交叉动态

> 时间窗口：2026-05-25 08:00 至 2026-05-26 08:00（Asia/Shanghai）。

## 1. MCP / 工具调用安全：审计字段已足够明确，应进入工程落地

- **动态**：Context7 查询 OpenAI API 与 MCP 文档显示，MCP 工具调用、审批请求、工具列表和调用结果均有结构化字段，可直接映射到审计日志。
- **关键字段**：
  - 工具选择：`tool_choice`、候选 `tools`、`allowed_tools`。
  - 函数调用：`tool_calls`、`function_call`、`name`、`arguments`、`call_id`、`function_call_output`。
  - MCP 调用：`server_label`、`mcp_list_tools`、`mcp_approval_request`、`mcp_approval_response`、`mcp_call`、`approval_request_id`、`status`、`error`、`output`。
- **风险点**：
  - 工具描述被提示注入污染，诱导模型选择高权限工具。
  - `arguments` 中夹带路径、URL、SQL、shell、云资源 ID 等敏感参数。
  - 工具 `output` 中包含二次提示注入，影响后续模型决策。
  - 审批只看工具名，不看参数和上下文。
- **来源**：OpenAI API Reference <https://developers.openai.com/api/reference/>；MCP Specification <https://modelcontextprotocol.io/specification/>。

## 2. 安全技能库：攻防两侧能力放大器

- **动态**：GitHub 上安全技能库、agent skill、自动化 playbook 项目持续活跃，部分项目将安全任务映射到 MITRE ATT&CK、NIST CSF、D3FEND、ATLAS 等框架。
- **技术要点**：
  - 防守价值：标准化告警分诊、资产枚举、日志查询、IOC 扩展、报告生成。
  - 攻击风险：低门槛拼接侦察、漏洞验证、社工文案、横向移动检查、云资源枚举。
  - 供应链风险：技能文件中可嵌入恶意命令、外链、提示注入或过宽权限声明。
- **处置建议**：
  - 技能库进入企业 agent 前必须做静态扫描和人工抽样。
  - 将技能拆分为只读、低危写入、高危执行三类权限域。
  - 高危技能必须绑定审批、隔离环境和结果复核。
- **来源**：GitHub Trending <https://github.com/trending>；MITRE ATT&CK <https://attack.mitre.org/>；MITRE ATLAS <https://atlas.mitre.org/>；NIST AI RMF <https://www.nist.gov/itl/ai-risk-management-framework>。

## 3. AI 安全评测：应覆盖“模型-工具-环境”闭环

- **动态**：CyberSec-Eval、厂商安全评测和学术研究的共同趋势，是把 LLM 风险从输出内容扩展到代码生成、漏洞利用、社工滥用和 agent 行为。
- **建议评测维度**：
  - 提示安全：jailbreak、角色扮演、上下文污染、长上下文绕过。
  - 工具安全：工具选择、参数生成、审批绕过、结果污染、跨 server 调用。
  - 代码安全：漏洞代码生成、补丁正确性、依赖风险、测试覆盖。
  - 滥用风险：钓鱼文案、恶意脚本、漏洞复现、资产扫描语句。
  - 防守能力：日志解析、IOC 归并、检测规则生成、误报解释。
- **来源**：CyberSec-Eval <https://github.com/meta-llama/PurpleLlama/tree/main/CybersecurityBenchmarks>；OpenAI API Reference <https://developers.openai.com/api/reference/>；MCP Specification <https://modelcontextprotocol.io/specification/>。
