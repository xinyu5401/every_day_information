# 大模型 + 网络安全交叉动态

## AI 生成钓鱼页面进入实际攻击链

- 时间：2026-05-22，具体时分待核实。
- 来源：
  - [CNCERT 银狐木马风险提示](https://www.cert.org.cn/publish/main/10/2026/20260522113326926111046/20260522113326926111046_.html)
- 技术细节：
  - 大模型可用于生成高仿登录页、通知页、下载页和本地化社工文案。
  - 攻击者可快速生成多套页面变体，规避基于固定模板或关键词的检测。
  - 与传统钓鱼相比，AI 参与降低了“页面质量”和“语言自然度”的门槛。
- 防御重点：
  - 对页面静态相似度的依赖要下降，转向多模态与行为侧证据：域名年龄、证书、跳转链、下载物、脚本行为、终端执行链。
  - SOC 需要把同源变体聚类，避免每个 AI 生成页面都触发孤立工单。

## 插件、工具调用与 MCP 成为代理安全主战场

- 时间：过去 24 小时 GitHub 日榜出现相关项目；协议/接口定义由 Context7 查询确认。
- 来源：
  - [GitHub Trending](https://github.com/trending?since=daily)
  - [anthropics/claude-plugins-official](https://github.com/anthropics/claude-plugins-official)
  - [MCP Specification](https://modelcontextprotocol.org/specification/)
  - [OpenAI Developers](https://developers.openai.com/)
  - Context7: OpenAI API 与 MCP 文档。
- 技术细节：
  - OpenAI API 侧：`instructions`、developer role、Structured Outputs、function tools schema、`strict: true` 可降低输出格式和工具参数失控风险。
  - MCP 侧：`readOnlyHint`、`openWorldHint`、`ToolChoice`、roots、sampling、用户同意原则可帮助表达工具权限边界。
  - 这些控制仍需配合运行时隔离、最小权限、出站网络控制、敏感操作二次确认和审计日志。
- 主要风险：
  - 工具描述投毒：攻击者通过文档、网页、README 或插件描述诱导模型选择危险工具。
  - Prompt injection：外部内容让代理忽略原有策略，读取或外传本地文件。
  - Supply chain：插件仓库、MCP server、依赖包和 OAuth app 都可能成为投毒点。
- 研判：
  - “能接工具”已经不是差异点，“能证明工具调用安全”会成为企业采用智能体的门槛。
