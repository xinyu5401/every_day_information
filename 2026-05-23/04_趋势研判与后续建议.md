# 趋势研判与后续建议

## 趋势研判

1. 扩散式 LLM 进入工程化视野  
   NVIDIA Nemotron-Labs Diffusion 显示文本生成架构正在从自回归单一路线扩展到扩散式路线。安全评测需要关注非自回归解码下的越狱稳定性、拒答一致性、中间状态泄露和安全过滤位置。

2. AI 生成钓鱼从概念验证转向真实攻击链  
   CNCERT 银狐木马提示表明，攻击者已经把 AI 用于页面与文案生产。未来钓鱼检测的关键不再是识别某个固定模板，而是识别快速变体背后的基础设施与行为链。

3. 代理工具调用安全会成为企业落地前置条件  
   GitHub 插件生态与 MCP/OpenAI 工具调用能力持续升温。结构化输出、工具 schema、权限提示和用户确认是必要控制，但还不足以覆盖供应链、越权访问和提示注入。

4. 高速模型降低滥用成本  
   高吞吐模型会让批量社工、批量漏洞说明生成、批量日志伪造和多语言变体生成更便宜。防御侧需要把调用速率、异常模式和内容聚类纳入检测。

## 后续建议

- 建立“LLM 架构变化”监测项：重点跟踪 diffusion LLM、混合推理模型、长上下文模型对现有安全评测的影响。
- 对内部代理平台强制落地工具调用审计：记录模型输入、工具选择、工具参数、执行结果、权限来源和用户确认状态。
- 将 AI 生成钓鱼纳入 SOC 用例：使用域名、证书、页面结构、下载物、终端行为和文本相似度联合聚类。
- 为高危漏洞增加“LLM 可利用性”评分：评估公开信息是否足够让模型生成复现步骤、检测脚本或利用链。
- 跟踪 MCP server 与插件供应链：对第三方插件、MCP server、OAuth app、依赖包进行准入审查和版本锁定。
- 对高速模型 API 建立滥用风控：监测异常调用量、批量模板生成、多语言社工文案和短时间大量相似输出。
- 每日固定复核 CNVD、CNCERT、FreeBuf、奇安信、深信服、绿盟科技；若官方站点未给出精确时分，报告中继续标注“待核实”。
