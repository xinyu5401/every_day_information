# 网络安全技术动态（2026-06-11）

## CISA KEV 到期窗口

### 1. Check Point Security Gateway `CVE-2026-50751` 进入 KEV 当日处置窗口

**入选理由**

该漏洞不是今天新披露，但 **CISA KEV 的 Due Date 为 2026-06-11**，命中今日处置窗口；同时它位于远程接入 / 边界网关面，优先级高于普通应用层漏洞。

**技术细节**

- 漏洞：`CVE-2026-50751`
- 组件：Check Point Security Gateway
- CISA KEV：
  - `Date Added`: **2026-06-08**
  - `Due Date`: **2026-06-11**
  - 窗口判断：**命中当日到期窗口**
- 漏洞类型：IKEv1 远程接入场景下的认证绕过 / improper authentication。
- 触发条件：公开资料显示，风险与以下部署条件相关：
  - 开启 IKEv1 远程接入；
  - 网关接受 legacy Remote Access clients；
  - 连接时未强制 machine certificate。

**影响与处置**

- 受影响资产以 VPN / 边界接入设备为主，若互联网暴露且仍保留 IKEv1 兼容配置，应按“边界设备优先”处理。
- 建议动作：
  - 立即核查是否仍启用 IKEv1 与 legacy client 兼容；
  - 能关则关，不能关则先提升到强制 machine certificate 的安全配置；
  - 同步按厂商指引升级或缓解，并对近期远程接入日志做异常认证排查。

**信息来源**

- https://www.cisa.gov/known-exploited-vulnerabilities-catalog
- https://support.checkpoint.com/results/sk/sk185033

## 2. Chromium V8 `CVE-2026-11645` 在野利用已进 KEV，影响面覆盖 Chrome / Edge / Opera

**入选理由**

Google 官方确认漏洞已被在野利用，CISA 已将其加入 KEV；且这是浏览器内核 V8 缺陷，影响面不止 Chrome，而是所有跟随 Chromium 的主流桌面浏览器。

**技术细节**

- 漏洞：`CVE-2026-11645`
- 组件：Chromium V8
- 类型：越界读写，可导致攻击者通过构造 HTML 页面在沙箱内执行任意代码。
- 官方状态：
  - Chrome 稳定版更新公告发布时间为 **2026-06-09**
  - Google 明确表示“已知存在在野利用”
- CISA KEV：
  - `Date Added`: **2026-06-09**
  - `Due Date`: **2026-06-23**
  - 窗口判断：未命中今日 / 未来 24 小时到期窗口，但属于新增 KEV 高优先级项目。

**影响与处置**

- 受影响对象包括但不限于 Google Chrome、Microsoft Edge、Opera 等 Chromium 系浏览器。
- 建议动作：
  - 桌面终端尽快升级到厂商已发布修复版本；
  - 对高风险终端加强网页内容隔离与下载/插件审查；
  - 对尚未统一打补丁的企业环境，优先处理高权限终端、对外运营岗位和可访问敏感 SaaS 的终端。

**信息来源**

- https://chromereleases.googleblog.com/2026/06/stable-channel-update-for-desktop_0153744567.html
- https://www.cisa.gov/known-exploited-vulnerabilities-catalog

## 3. Jenkins Core 修复 `CVE-2026-53435`，低门槛权限可升级为任意用户请求伪造与控制器文件读取

**入选理由**

Jenkins 官方安全公告于 **2026-06-10** 发布，受影响对象是 Jenkins core 本体而非单个插件；漏洞利用后果包含任意用户模拟请求、Script Console 滥用链和控制器文件读取，属于高价值 CI/CD 基础设施风险。

**技术细节**

- 漏洞：`CVE-2026-53435` / `SECURITY-3707`
- 严重性：High
- 受影响版本：
  - Jenkins **2.567 及更早版本**
  - Jenkins LTS **2.555.2 及更早版本**
- 修复版本：
  - Jenkins **2.568**
  - Jenkins LTS **2.555.3**
- 机理：
  - 攻击面来自 Jenkins 在 `config.xml` 提交路径上的反序列化处理。
  - 攻击者可让 Jenkins 反序列化核心或插件内任意类型，再借此获得后续 HTTP 请求处理能力。
- 利用前提：
  - 需要 `Overall/Read`；
  - 还需至少满足“拥有用户账户”或“拥有可 `POST config.xml` 的相关配置权限”之一。
- 公开后果：
  - 可模拟任意用户发起 HTTP 请求；
  - 最高可借 Script Console 走到任意代码执行；
  - 可读取 Jenkins controller 任意文件。

**影响与处置**

- 这是典型“低到中权限起步，向控制平面跃迁”的 CI/CD 风险，不应只按普通本地提权处理。
- 建议动作：
  - 尽快升级到 2.568 / 2.555.3；
  - 复核谁具备 `Overall/Read`、`Item/Configure`、`View/Configure`、`Agent/Configure` 等权限；
  - 检查近期 `config.xml` 相关提交与 Script Console 调用日志。

**信息来源**

- https://www.jenkins.io/security/advisory/2026-06-10/

## 4. Ivanti Sentry 爆出 `CVE-2026-10520` / `CVE-2026-10523`，未授权即可打到 root RCE 或管理员创建

**入选理由**

Ivanti 于 **2026-06-09** 发布官方安全公告，Rapid7 同步跟进技术解读。目标产品是企业移动流量网关类边界设备，且两枚漏洞都支持远程未授权利用，需作为高优先级边界风险关注。

**技术细节**

- 组件：Ivanti Sentry（原 MobileIron Sentry）
- 漏洞：
  - `CVE-2026-10520`：OS Command Injection，CVSS **10.0**
  - `CVE-2026-10523`：Authentication Bypass，CVSS **9.9**
- 官方公开受影响范围摘要：
  - `CVE-2026-10520` 影响修复前版本，官方摘要点名修复版本线为 **R10.5.2 / R10.6.2 / R10.7.1**
- 利用后果：
  - `CVE-2026-10520` 可让远程未授权攻击者以 **root** 权限执行命令；
  - `CVE-2026-10523` 可让远程未授权攻击者创建任意管理员账户并获得完整管理权限。
- Rapid7 转述厂商表态：披露时厂商称**尚未发现客户被利用**。

**影响与处置**

- 该产品位于移动设备到后端企业系统之间，若互联网暴露则优先级接近传统 VPN / WAF / 邮件网关。
- 建议动作：
  - 立即核对版本是否已到 R10.5.2 / R10.6.2 / R10.7.1 或更高；
  - 对管理账户新增、异常 shell 执行、配置变更和外联日志做回溯；
  - 若短期无法升级，至少应临时缩减暴露面并收紧管理平面访问。

**信息来源**

- https://hub.ivanti.com/s/article/Security-Advisory-Ivanti-Sentry-CVE-2026-10520-CVE-2026-10523?language=en_US
- https://www.rapid7.com/blog/post/etr-cve-2026-10520-cve-2026-10523-multiple-critical-vulnerabilities-affecting-ivanti-sentry/
