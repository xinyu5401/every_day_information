# 昨日焦点专题（2026-06-11）

## Claude Fable 5 / Mythos 5：Anthropic 把前沿模型能力发布与“分级放权”安全运营绑定

Anthropic 在 **2026-06-09** 发布 `Claude Fable 5` 与 `Claude Mythos 5`。这次不是单纯升级旗舰模型，而是把“公开可用版”和“受控放开版”同时推出，说明头部厂商正在把高能力模型按风险域做制度化切分。

### 为什么值得单独成篇

- 这是过去 36 小时内最重要的头部模型发布之一。
- Mythos 5 明确面向网络安全防守与关键基础设施合作方，意味着“网络安全能力受限开放”已从研究姿态转入产品化分发。
- Fable 5 对危险主题启用分类器并回退到 `Opus 4.8`，说明模型安全不再只靠拒答，而是开始引入**动态模型路由**。

### 发布结构

- `Claude Fable 5`
  - 官方定义为可普遍提供的最高能力 Claude。
  - 面向软件工程、长程知识工作、视觉推理、多步自主执行。
  - 通过 Claude API 与消费型 / 企业型产品体系分阶段上线。
- `Claude Mythos 5`
  - 与 Fable 5 为同一底座。
  - 在部分网络安全方向移除限制。
  - 当前仅向 Project Glasswing 合作方开放，后续再扩展 trusted access。

### 这次发布真正改变了什么

**1. 安全策略从“统一能力面”变成“按风险域拆模型体验”**

Anthropic 明确表示，Fable 5 在高风险主题会回退到 `Opus 4.8`。这意味着生产环境中的“实际响应模型”可能与用户指定模型不同。对外部接入方来说，模型 ID、响应链路、拒答与回退日志都必须进入可观测体系，否则很难解释行为波动。

**2. 高能力网络安全模型开始走“受控放权”分发**

Mythos 5 不直接全面开放，而是先给防守方和关键基础设施组织。这种做法和传统高危工具的受限分发更接近，后续值得持续观察：

- access policy 是否细化到组织类型、用途与审计要求；
- system card / safety card 是否补充越权、双用途和对抗绕过细节；
- 是否要求调用日志、人工审批或更高强度身份校验。

**3. 价格与配额策略也在配合能力分层**

- Fable 5 / Mythos 5 定价均为 **$10 / 百万输入 token，$50 / 百万输出 token**。
- Fable 5 在订阅方案中的“包含期”只到 **2026-06-22**。
- **2026-06-23** 起，如继续使用，需要 usage credits。

这说明 Anthropic 还在观察容量与风险，不愿立即把该模型永久并入所有订阅层。

### 对代码 Agent / 安全 Agent 的现实含义

- 如果业务依赖长程自治、代码迁移、复杂工具调用，Fable 5 的上线会显著提升任务完成上限。
- 但只要请求落在高风险域，实际可能触发 Opus 4.8 回退，导致能力、拒答率、步骤数和延迟分布变化。
- 如果组织希望用其做安全研究、漏洞分析或攻防辅助，公开版 Fable 5 与受控版 Mythos 5 的能力边界不能混用评估。

### 建议立即核对

- API 是否显式指定 `claude-fable-5`。
- 监控里是否记录最终服务模型、回退事件、拒答类型。
- 6 月 23 日后的 usage credits 与速率预算是否已单独评估。
- 安全、研发、普通业务是否使用同一模型路由与同一权限配置。
