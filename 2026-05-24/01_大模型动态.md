### LCGuard：面向多智能体 KV 共享的潜在通信防护

- 来源：[arXiv:2605.22786](https://arxiv.org/abs/2605.22786)
- 提交时间：2026-05-21 17:42:12 UTC。
- 技术细节：论文将多智能体系统中的共享 KV cache 视为 latent working memory，提出在跨 agent 传输 cache artifacts 前做表示层变换。
- 安全问题：KV cache 可能携带上下文输入、中间推理状态和 agent 专属信息，形成不可见的敏感信息传播通道。
- 方法：通过对抗训练定义“可重构敏感信息”为泄露指标，降低攻击者 decoder 从共享 cache 中恢复 agent-specific sensitive inputs 的成功率，同时维持任务语义。
- 趋势判断：长上下文、KV cache 复用、agent 间隐式通信会改变现有数据泄露评测边界，需把“非文本中间状态”纳入安全审计。

### HarnessAPI：从单一 skill 源生成 HTTP 与 MCP 工具

- 来源：[arXiv:2605.22733](https://arxiv.org/abs/2605.22733)
- 提交时间：2026-05-21 17:03:44 UTC。
- 技术细节：以 typed skill folder 为单一事实源，从 `handler.py` 与 Pydantic schemas 自动派生 SSE HTTP endpoint、OpenAPI/Swagger UI 和 MCP tool。
- 工程价值：减少 FastAPI + FastMCP 双栈手工维护的样板代码，降低工具 schema 漂移。
- 安全关注：一处 skill 定义同时生成对人类和 agent 暴露的接口，任何 schema、鉴权或输入验证缺陷会同步扩散到 HTTP 与 MCP 两个入口。

### SkillWeave：模块化 skillpacks 提升 LLM 专业能力

- 来源：[arXiv:2605.22205](https://arxiv.org/abs/2605.22205)
- 提交时间：2026-05-21 09:12:20 UTC。
- 技术细节：将通用模型能力拆分为轻量 domain-specific delta modules，并通过 SkillZip 压缩为可部署格式。
- 性能主张：9B SkillWeave 模型在多任务与 agentic benchmarks 上超过若干 baseline，并实现最高 4x speedup。
- 安全关注：skillpack 会成为模型能力供应链的一部分，需要评估 skillpack 来源、版本、能力边界、恶意能力注入与回滚机制。
