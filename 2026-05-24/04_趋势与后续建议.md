# 2026-05-24 趋势与后续建议

## 趋势判断

### 1. Agent 安全从 prompt 层转向执行层

过去 24 小时的异常热度集中在 GitHub Trending 的 agent skills、插件、MCP 与代码图谱项目，以及 CNVD 众测平台的智能体应用安全众测持续活跃。安全边界正在从“模型是否回答危险内容”转向“模型是否能调用高权限工具完成危险动作”。

### 2. MCP 与插件供应链成为新准入对象

MCP 的 `tools/list` 和 `tools/call` 让工具发现与执行具备统一接口，但也意味着工具描述、schema、权限与运行结果都需要可审计。第三方 MCP server、skills、OAuth app 和依赖包应被视为生产供应链组件。

### 3. LLM 架构变化会削弱旧评测覆盖

LCGuard 指出的 KV cache 泄露问题说明，多智能体系统的敏感信息不一定以文本形式出现。Diffusion LLM、混合推理模型、长上下文模型、KV cache 共享和 skillpack 模块化都会改变泄露、越权和后门评测方式。

### 4. 官方测试活动推动 AI 安全从概念走向定级闭环

CNCERT/CNVD 相关活动已经覆盖大模型护栏、智能体恶意操作、漏洞智能挖掘、告警降噪和 AI 生成内容检测。这会推动国内厂商建立可复现、可定级、可修复、可复测的大模型安全流程。

## 后续建议

1. 建立“LLM 架构变化”监测项：重点跟踪 diffusion LLM、混合推理模型、长上下文模型、KV cache 共享、模块化 skillpack 对现有安全评测的影响。
2. 对内部代理平台强制落地工具调用审计：记录模型输入、候选工具、工具选择、工具参数、执行结果、权限来源和用户确认状态。
3. 将 AI 生成钓鱼纳入 SOC 用例：使用域名、证书、页面结构、下载物、终端行为、文本相似度、水印和内容凭证联合聚类。
4. 为高危漏洞增加“LLM 可利用性”评分：评估公开信息是否足够让模型生成复现步骤、检测脚本或利用链。
5. 跟踪 MCP server 与插件供应链：对第三方插件、MCP server、OAuth app、依赖包进行准入审查、权限评估和版本锁定。
6. 对高速模型 API 建立滥用风控：监测异常调用量、批量模板生成、多语言社工文案和短时间大量相似输出。
7. 每日固定复核 CNVD、CNCERT、FreeBuf、奇安信、深信服、绿盟科技；若官方站点未给出精确时分，报告中继续标注“待核实”。
8. 对浏览器与终端类 agent 工具启用隔离 profile 和沙箱；禁止默认访问生产账号、生产凭据和真实客户数据。
9. 将 AI agent skills 仓库纳入安全评审，重点检查危险命令、凭据读取、漏洞复现、外联目标和数据持久化行为。
10. 对模型 alias 和推理模式做日志标准化：记录真实模型、版本、thinking/non-thinking 模式、上下文窗口和工具能力开关，避免安全评测基线漂移。
