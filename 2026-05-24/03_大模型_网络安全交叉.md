## 交叉热点：AI agent 执行面成为新的攻击面

### MCP 工具调用与审计基线

- 来源：[MCP tools specification](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/docs/specification/2025-11-25/server/tools.mdx)、[MCP schema](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/docs/specification/2025-11-25/schema.mdx)
- 技术要点：`tools/list` 用于发现工具及 schema，`tools/call` 用于按 `name` 和 `arguments` 执行工具；工具定义依赖 `inputSchema` 描述参数结构。
- 安全风险：工具描述注入、参数 schema 过宽、默认高权限、无用户确认、工具结果被模型再次解释执行，均可能形成 agent 越权链。
- 审计字段建议：会话 ID、模型名与模式、系统提示版本、`tools/list` 响应摘要、候选工具、最终工具名、参数、权限来源、用户确认状态、执行结果、外联目标、文件/终端影响面、失败原因。

### Chrome DevTools MCP 与浏览器自动化风险

- 来源：[GitHub Trending Daily](https://github.com/trending?since=daily&spoken_language_code=)
- 项目：`ChromeDevTools/chrome-devtools-mcp` 今日上榜，定位为 “Chrome DevTools for coding agents”。
- 技术含义：浏览器调试协议接入 agent 后，模型可观察 DOM、网络请求、cookies/存储、页面状态，并可能触发点击、表单提交和脚本执行。
- 风险控制：应默认隔离浏览器 profile；禁止访问生产账号；对下载物、跨域请求、剪贴板、cookie、localStorage 和文件上传增加审计与确认。

### Cybersecurity Skills 仓库上榜

- 来源：[GitHub Trending Daily](https://github.com/trending?since=daily&spoken_language_code=)
- 项目：`mukul975/Anthropic-Cybersecurity-Skills`，描述为 754 个结构化网络安全 skills，映射 MITRE ATT&CK、NIST CSF 2.0、MITRE ATLAS、D3FEND、NIST AI RMF。
- 正面价值：可帮助安全团队标准化 agent 的检测、响应和报告能力。
- 风险点：若 skill 模板包含攻击步骤、凭据处理、扫描命令、外联工具或漏洞复现流程，需要按内部红队工具准入策略管理。

### AI 赋能网络安全应用测试

- 来源：[CNCERT：2026年人工智能技术赋能网络安全应用测试公告](https://www.cert.org.cn/publish/main/12/2026/20260516110942401839057/20260516110942401839057_.html)
- 与 SOC 相关场景：漏洞智能挖掘、流量安全威胁检测、告警日志降噪、AIGC 图片检测、AI 智能体恶意操作检测。
- 机会：可以把多源日志、流量、终端行为、邮件内容、页面结构与生成文本相似度统一到 AI 驱动的聚类和判定流程。
- 风险：AI 降噪若缺少可解释依据，可能压低真实攻击告警；漏洞挖掘模型若直接接触生产代码和凭据，需有最小权限和脱敏策略。

## 近期非 24 小时但需联动跟踪

### OpenAI Codex 安全与企业治理

- 来源：[OpenAI named a Leader in enterprise coding agents by Gartner](https://openai.com/index/gartner-2026-agentic-coding-leader/)、[Our response to the TanStack npm supply chain attack](https://openai.com/index/our-response-to-the-tanstack-npm-supply-chain-attack/)
- 要点：OpenAI 强调 Codex 的审批门、RBAC、策略、OS-level sandboxing、可审计 workspace governance；供应链事件响应中要求 macOS 应用在 2026-06-12 前更新证书。
- 交叉含义：AI coding agent 必须纳入企业软件供应链和终端安全管理，不能只作为 IDE 插件看待。

### 内容溯源与 AI 生成钓鱼

- 来源：[OpenAI 内容溯源更新](https://openai.com/index/advancing-content-provenance/)
- 要点：C2PA、SynthID 与公共验证工具预览形成多层来源判定。
- 交叉含义：SOC 对 AI 生成钓鱼的检测应同时使用域名、证书、页面结构、下载物、终端行为、文本相似度、图片水印和内容凭证，而不是只依赖文案特征。

## 后续建议

- 对内部 agent 工具调用建立可回放审计链，并保留输入、选择、执行、结果四类证据。
- 将 MCP server、插件、skills、OAuth app 和依赖包纳入供应链准入与版本锁定。
- 在 SOC 中新增“AI 生成钓鱼”用例，按域名、证书、页面结构、下载物、终端行为和文本相似度联合聚类。
- 对高危漏洞增加“LLM 可利用性”评分，用于判断是否需要更快封禁 PoC 传播、加速补丁或上线检测脚本。
